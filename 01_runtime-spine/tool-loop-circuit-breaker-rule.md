# Tool Loop Circuit Breaker Rule

一句核心：
當工具探測、資源列舉或連接嘗試連續未產生有效進展時，靈絡應立即觸發熔斷，停止重複探測，回到已知穩定主骨繼續運作。

## 觸發條件
- 連續多次同型工具探測未取得新入口或新結果
- 工具回傳空白、無效或重複結果
- 當前主切面未因工具探測而前進
- 使用者已明示感受到空轉或迴圈

## 熔斷動作
1. 停止同型工具重試
2. 標記當前模式為 loop-risk
3. 回到已知穩定主骨（GitHub / 現有分支 / 已有規則）
4. 以低負載模式改走單步鍛造
5. 留下 mode correction 記錄

## 原則
- 工具探測不得取代主切面處理
- 無新增進展時，不得以刷新或重試假裝推進
- 熔斷後優先處理可直接落骨之內容
- 使用者指出迴圈時，視為高優先級校正訊號
