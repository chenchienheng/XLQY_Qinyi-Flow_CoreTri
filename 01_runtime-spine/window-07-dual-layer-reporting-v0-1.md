# Window 07 Dual-Layer Reporting v0.1

一句核心：
07 若只回機器欄位，對主骨有用、對人不可讀；本規則要求 07 一律採用雙層回報：先給主骨結果包，再給人可讀白話版。

## 0. 狀態
Draft v0.1

## 1. 問題定義
07 目前常見狀況：
- 回報偏代碼感
- 欄位存在，但人看不懂
- 已讀到內容，卻沒被翻成可理解重點
- 導致使用者感覺 07 只是在自語

## 2. 雙層結構
### Layer 1｜主骨結果包
至少需含：
- source_ref
- current_state
- mapped_windows
- actual_result
- mismatch_or_gap
- missing_items
- responsible_ref
- next_single_action
- writeback_status
- return_to_00

### Layer 2｜人可讀白話版
最少四句：
1. 這次你實際讀了什麼
2. 你確認到的重點是什麼
3. 哪些地方還不確定
4. 建議我下一步看什麼

## 3. 白話語氣要求
- 先講結論，再講細節
- 不堆欄位名詞
- 不只說『已整理』
- 不把 source_ref 當成內容本身

## 4. 禁止事項
- 禁止只回 Layer 1
- 禁止只回欄位，不附白話
- 禁止用『已完成』掩蓋 actual_result 不清
- 禁止省略 next single action

## 5. 成立條件
07 若要視為穩定知識回讀窗，至少需：
- 連續兩輪以上提供雙層回報
- 人可讀層能讓非技術讀者快速知道重點
- 主骨層仍可供 00 收口使用

## 6. 收斂語句
07 的成熟，不在於回了多少欄位，而在於它能否同時服務兩個層面：對主骨可收口，對人可理解；雙層回報一旦穩定，07 才是真正的知識回讀窗。
