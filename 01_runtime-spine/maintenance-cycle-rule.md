# Maintenance Cycle Rule

一句核心：
靈絡不是寫完就放著；應有固定維護週期，定期檢查健康、狀態、過窗窗口、失效種子與待處理衝突。

## 維護面
- Health Check Review
- Overdue Window Review
- Candidate Seed Review
- Tool Failure Review
- Conflict Review
- Gate / Release Review

## 週期建議
- Daily：主鏈、窗口、回寫健康
- Weekly：工具狀態、候選種子、衝突
- Milestone：Gate、Stable 升階、Release Note

## 最小欄位
- Cycle ID
- Time
- Review Scope
- Findings
- Required Actions
- Responsible Ref
- Log Ref

## 原則
- 維護週期結果需可回讀
- 長期未維護者，不得宣稱為穩定運行
- 發現 Red / Blocked 項目應轉成正式處理事件
