# Low Load Execution Mode Rule

一句核心：
當工具鏈不穩、外部接面未明或主鏈需止散時，靈絡應切入低負載執行模式，以單步、單骨、單驗證方式持續前進。

## 適用情境
- 工具連接不穩
- 外部盤工具未定
- 使用者要求先收斂再前進
- 出現 loop-risk 或過度探測跡象

## 執行方式
- 一次只新增一個主骨或一個實例
- 每次寫入後立即驗證
- 不同層不並行擴張
- 優先在 GitHub 主本分支內完成鍛造
- 外部工具只作後續候選，不即時依賴

## 原則
- 先穩再快
- 先主鏈再外環
- 先可驗再擴張
- 先止散再接器官
