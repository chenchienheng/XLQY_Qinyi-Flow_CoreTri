# Supersession Rule

一句核心：
新規則、新版本或新窗口取代舊成立面時，必須明確標示 supersedes 關係；未標示者，只能並存，不得自稱已正式取代。

## 適用對象
- Rule -> Rule
- Version -> Version
- Window -> Window
- Project Card -> Project Card
- Mirror Spec -> Mirror Spec

## 最小欄位
- Supersession ID
- Source Ref
- Target Ref
- Time
- Reason
- Responsible Ref
- Log Ref

## 規則
- 被取代內容不得直接消失，應保留引用入口
- supersedes 關係需可對回版本、窗口或主寫位
- 若僅局部取代，需標示 scope
- 無 supersedes 記錄者，不得宣稱完成替換
