# Stable Promotion Rule

一句核心：
Seed 層內容要升為 Stable 層，不能只因可用或急需；必須證明其已可回讀、可續接、可回退、且衝突已受控。

## 升階條件
- 已有核心句與作用範圍
- 已有 Responsible Board 與 Primary Write
- 已有 Writeback Contract
- 已有 Snapshot 與 Recovery Point
- 已有 Conflict Resolution 或無衝突說明
- 已有至少一次可追讀的運作記錄

## 結果
- Promote：可升為 Stable
- Hold：仍留 Seed 層待補
- Reject：不符合升階條件

## 原則
- 可運行不等於可升階
- Stable 必須能作穩定參照
- 升階結果需附時間、版本與理由
- 未升階前不得冒充穩定母法
