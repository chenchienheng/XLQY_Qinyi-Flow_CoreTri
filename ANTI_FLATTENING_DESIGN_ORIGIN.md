## 2. Public Architecture Definition

XuanLing's design origin is not product-first, feature-first, or automation-first.

It begins from an anti-flattening principle:

```text
people, relationships, tools, systems, love, tasks, and models must not be compressed into replaceable nodes, resources, or execution objects.
```

Chinese:

```text
翾靈的設計起點不是產品優先、功能優先或自動化優先，而是反壓扁原則：人、關係、工具、系統、愛、任務與模型，都不能被壓縮成可替換節點、資源或執行物。
```

Therefore, XuanLing prioritizes before capability expansion:

- boundary
- responsibility
- state distinction
- authorization
- evidence
- return

---

## 3. Why This Explains XuanLing Rules

This origin explains why XuanLing repeatedly generates rules such as:

```text
User ≠ Node
Qinyi ≠ XuanLing
Candidate ≠ Approved
Return Packet ≠ Closeout
Tool capability ≠ authorization
Resonance ≠ evidence

```

These are not decorative principles.
They are anti-flattening protections.

---

## 4. Design Difference

Common product logic often begins with:

```text
need
→ feature
→ user flow
→ technical architecture
→ productization
```

XuanLing's design origin begins closer to:

```text
life pressure
→ relationship boundary
→ responsibility position
→ tool sovereignty
→ error return
→ judgment generation
→ architecture
```

This does not make XuanLing complete.
It explains why its architecture prioritizes boundary and responsibility before expansion.

## 5. Public / Private Split

Public layer may include:

- anti-flattening design origin
- human-boundary first principle
- relationship / tool / task / model anti-reduction rules
- private-context boundary policy
- candidate architecture interpretation

Private living context may support the architecture only as:

- private living source
- life-course resonance
- low-pressure relationship boundary case
- non-public support material

Private living context must not be promoted into:

- system module
- node
- governance source
- public proof object
- authority role
- doctrine evidence

## 6. Private Boundary Sentence

Internal-safe formulation:

```text
A named private person is not a XuanLing tool, node, or governance role. A private relationship may function only as a private living source showing how love without possession, importance without control, and closeness without pressure became concrete for the user.
```

This sentence must not be used to make the private person responsible for the architecture.
