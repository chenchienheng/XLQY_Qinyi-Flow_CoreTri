# External Node On-Chain Specification

> Durable specification for onboarding external ecosystem nodes into the XLEN / Xuanling runtime.
> Purpose: ensure external tools, platforms, boards, model surfaces, and publication surfaces become chain-bound nodes instead of uncontrolled dependencies.

---

## 0. Core Reading

External nodes should not be treated as:

- isolated tools
- uncontrolled dependencies
- opaque black-box integrations
- sovereign centers
- proof of completion by mere connection

They should be treated as:

- role-bound nodes
- replaceable ecosystem members
- chain-aware execution or interaction surfaces
- evidence-bearing carriers when logs or artifacts exist
- returnable participants in the shared continuity order

---

## 1. Topology-Sphere Rule

The broader XuanLing topology is not a flat tool map.

Tools, models, documents, chats, images, boards, clouds, GitHub, Zenodo, and workflow systems are sphere-surface nodes.
They are not the task body itself.

```yaml
Topology_Sphere_Node:
  center: invariant core / value key / return authority
  inner_shell: DCP / CoreTri / XuanLing
  operating_shell: Qinyi / MotherTree or PulseMesh / model invocation discipline
  carrier_shell: GitHub / Drive / M365 / boards / chats / issues / files
  public_shell: Zenodo / web / public whitepaper / external interface
```

A node joins the topology only when it can be read by source, carrier, authority, surface, and return.

---

## 2. On-Chain Conditions for External Nodes

An external node is considered on-chain only when:

1. **role is defined**
2. **owner_window is assigned**
3. **input/output is bounded**
4. **writeback path exists**
5. **replacement is possible**
6. **evidence or gap state is visible**
7. **authority ring is known**

## 5. Failure Mode Rule

Each node must define:

- what happens when it fails
- fallback path
- alternative node
- whether work can continue without it
- whether state is preserved elsewhere

---

## 6. Anti-Dependency Rule

A node is unsafe if:

- it becomes irreplaceable
- it holds critical state without writeback
- it controls routing without visibility
- it claims execution without evidence
- it shifts authority from user / 00 into the tool itself

---

## 7. Tri-Coupled Node Test

A node is not fully XuanLing-bound unless it passes three checks:

```yaml
Tri_Coupled_Node_Test:
  Xiangyi_相依:
    ask: what does this node depend on and what depends on it?
  Hucun_互存:
    ask: how does this node cooperate without collapsing into another role?
  Xulian_續鏈:
    ask: how does output persist, return, and remain resumable?
```

If any one is missing, the node remains partial / candidate.
