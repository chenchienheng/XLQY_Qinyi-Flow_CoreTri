# SEMANTIC FIREWALL｜Open XuanLing Core

Status: Candidate / Semantic Firewall / Not Approved / No Runtime

## Core

Semantic Firewall prevents meaning drift and state pollution. It keeps capability, identity, authority, state, and action separate.

## Core Rules

- Capability != Permission.
- Access != Writeback.
- Assistant != Authority.
- Agent != Human Approval.
- Persona Surface != Human Identity.
- Tool Connection != System Authorization.
- Public-safe != Public-approved.
- Source Record != Final Decision.
- Return Packet != Closeout.

## State Rules

- Candidate != Approved.
- Approved != Runtime.
- BuildReady != Runtime.
- Review != Human Approval.

## Adapter Rules

- Core != Adapter.
- Adapter != Runtime.
- Runtime != Permission.
- Platform availability != Governance approval.

## Not To Claim

Do not use words that imply approval, action, runtime, identity, or permission unless the corresponding authority and gate are present.
