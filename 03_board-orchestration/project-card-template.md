# Project Card Template

一句核心：
主案要能續行，需先有最小主案卡；沒有主案卡，事件再多也只是散點，不能成案。

## 最小欄位
- Project Key
- Project Name
- Core Sentence
- Primary Slice
- Responsible Board
- Primary Write Target
- Mirror Target
- Time Window
- Current Status
- Next Action
- Writeback Condition
- Log Ref

## 用途
- 建立主案識別
- 統一回寫口
- 對齊盤位與工具
- 提供後續 handoff 依據

## 原則
- 每一主案至少一張卡
- 卡片變更需更新狀態與下一步
- 無 Project Key 者，不得長期佔用主鏈
