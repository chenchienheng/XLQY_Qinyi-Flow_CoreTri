# GitHub Native Board Workflow — Seed v0.1

一句核心：
靈絡若以 GitHub 原生盤運作，應先以固定工作流處理進件、抽骨、盤面更新、任務承接、回寫與日誌，不依賴外部盤工具作節奏主導。

## Workflow
1. Intake enters chat context
2. Extract core into GitHub source file
3. Update GitHub Native Board item
4. If action needed, append Task Follow-up item
5. If state changed, append Log Entry Ref
6. If external mirror is needed, sync after GitHub writeback

## 適用面
- Governance Core tracking
- Blocker tracking
- Next Action queue
- Handoff routing
- Seed status review

## 原則
- 先 GitHub，後鏡像
- 先抽骨，後盤面
- 先回寫，後展示
- 盤面更新需保留 source ref、writeback position 與 log entry ref

## 狀態
Seed v0.1
