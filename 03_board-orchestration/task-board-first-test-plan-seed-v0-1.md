# Task Board First Test Plan — Seed v0.1

一句核心：
② 任務盤首輪實測應以最低負載驗證其是否能承接 next action、責代追蹤與 handoff 後續，而不是一次塞滿所有任務管理功能。

## Test Plan ID
- TBT-001

## Candidate Tool
- ClickUp

## 測試目標
- 是否能作為 ② 任務盤首輪候選
- 是否能承接最小 next action 清單
- 是否能標示 Responsible Board / Responsible Ref
- 是否能與 ⓪ 公盤與 GitHub 主本形成 handoff 路徑

## 最小測項
1. 建立一個 Seed 任務盤頁面或清單
2. 放入至少三個任務項：BR-02 follow-up、BR-03 current test、writeback follow-up
3. 標示 Responsible Board、Next Action、Status
4. 測試是否能從公盤入口清楚 handoff 至任務盤
5. 不進行 rule 主寫，只作任務承接與追蹤

## 成功條件
- 可視為 ② 任務盤首輪可用工具
- 不與 GitHub 主寫位衝突
- 能承接最小任務追蹤與 handoff follow-up

## Notes
- 首輪實測通過後，才可討論是否升為任務盤主工具
- 若 ClickUp 首測不理想，再切換 Asana 作替代測試

## 狀態
Seed v0.1
