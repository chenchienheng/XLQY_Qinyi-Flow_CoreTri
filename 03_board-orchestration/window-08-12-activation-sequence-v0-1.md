# Window 08-12 Activation Sequence v0.1

一句核心：
08–12 不應同時亂開；其正確啟動方式，應依承載、對位、整合、生成、世界場五階段逐步點亮，避免高位願景先於低位收口。

## 0. 狀態
Draft v0.1

## 1. 啟動順序
1. 08｜Agent Runtime Board
2. 09｜UI / Surface Board
3. 10｜Model Family Integration Board
4. 11｜Lingluo Autonomous Network Board
5. 12｜Cluster Neural Open Runtime Board

## 2. 啟動理由
### Phase 1｜08 先行
- 先有 agent-style 執行面
- 驗證長鏈任務可否被主骨調度
- 不求大，只求最小可運行 surface

### Phase 2｜09 接上
- 將 08 的內部執行結果外顯化
- 建立 UI / surface / interaction 面
- 驗證外顯面是否服從母本與收口

### Phase 3｜10 整合模型
- 把多模型家族降格成角色模組
- 不讓品牌直接進骨
- 建立 model adapter contract 的實測場

### Phase 4｜11 自成網絡
- 當 08–10 穩定後，開始降低單一第三方依賴
- 建立 bridge nodes、readback surfaces、自承接條件

### Phase 5｜12 世界場承載
- 僅在前四階段皆已有可行域後才啟動
- 承接 cluster neural / open runtime / worldfield 願景
- 不得先於 08–11 完成前被誤當成完成態

## 3. 各階段最小啟動條件
### 08 啟動條件
- 已有 result packet schema
- 已有 return_to_00 路由
- 至少一條 agent runtime path 可驗證

### 09 啟動條件
- 08 已產出最小 surface path
- main / seed / runtime surface 已可對位
- UI 不反客為主

### 10 啟動條件
- model adapter contract 已成立
- 至少兩個模型家族能以 role 進場
- verify / writeback / return_to_00 可被檢查

### 11 啟動條件
- 08–10 已能形成穩定局部閉環
- bridge nodes / autonomous readback conditions 已明確
- 不依附單一供應商的最低條件已可描述

### 12 啟動條件
- feasible domain 已被 00 裁定至少為 candidate
- 08 / 09 / 10 / 11 皆已有穩定結果包樣本
- worldfield 不再只是敘事

## 4. 禁止事項
- 禁止 12 先於 08–11 被高調宣告
- 禁止 10 在未有 adapter contract 前就談模型整合
- 禁止 09 只做 UI 漂亮、不對回主骨
- 禁止 08 把 agent 自由擴張當成成熟

## 5. 收斂語句
08–12 的正確啟動，不是一起打開，而是按主骨承受力逐階段點亮；先讓 agent 能服從，讓 surface 能外顯，讓模型能降格，讓網絡能自治，最後才談世界場。
