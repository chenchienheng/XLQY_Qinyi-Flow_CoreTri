# Intake Triage Rule

一句核心：
進件後不得直接佔用主鏈；必須先經分流判定，決定其進入主切面、候選種子、暫存或退回補件。

## 分流結果
- Main Slice：進入主切面候選
- Candidate Seed：掛入候選種子隊列
- Hold：暫存待補資訊
- Return：退回補件或重述

## 判定依據
- 規則含量是否足夠
- 是否已有明確核心句
- 是否存在時間窗
- 是否已有建議盤位
- 是否超出目前可行域

## 規則
- 未經分流，不得直接進入 Active
- 高密度輸入可先進 Candidate Seed，再挑主切面
- 缺核心句或時間窗者，優先進 Hold 或 Return
- 分流結果需留下 log 與來源引用
