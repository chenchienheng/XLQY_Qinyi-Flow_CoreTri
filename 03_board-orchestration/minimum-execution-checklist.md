# Minimum Execution Checklist

一句核心：
任何事件要正式進入靈絡運作前，至少需通過最小執行檢查；未通過者，只能列為候選或暫態。

## 檢查項
- 是否已判定主切面
- 是否已有 Responsible Board
- 是否已有 Primary Write Target
- 是否已有 Time Window
- 是否已有 Writeback Condition
- 是否需啟用工具，若需是否已指定主工具

## 結果分類
- Pass：可正式進入運作
- Partial：僅可進入候選或待補
- Fail：不得視為成立

## 原則
- 先過檢查，再進運作
- 檢查結果需可回讀
- Partial 不得冒充 Pass
- Fail 需標示缺口位置
