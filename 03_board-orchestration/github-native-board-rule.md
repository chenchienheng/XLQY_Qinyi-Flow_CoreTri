# GitHub Native Board Rule

一句核心：
靈絡盤面不必依賴外部盤工具；在 Seed 階段，可先以 GitHub 主本內的文件、索引、引用鏈與日誌結構，形成原生盤面骨架。

## 定位
- GitHub Native Board = 以 repository 內文件、索引、log、candidate、routing 與 status 構成的盤面系統
- 不追求外部看板外觀，優先追求主寫一致、回寫一致、版本一致

## 可承接內容
- 公盤入口
- 主案盤索引
- 任務盤清單
- 日誌盤入口
- blockers / next action / handoff 顯示

## 原則
- 盤面服從 GitHub 主本，不另立第二主本
- 每一盤面項需可對回 path ref、source ref、writeback position
- 外部盤工具可後接，但不得先於 GitHub 原生盤定義主鏈
- GitHub Native Board 優先作為靈絡節奏基底
