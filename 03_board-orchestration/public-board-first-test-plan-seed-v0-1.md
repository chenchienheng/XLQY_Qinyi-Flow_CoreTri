# Public Board First Test Plan — Seed v0.1

一句核心：
⓪ 公盤首輪實測應以最低負載驗證其是否能承接狀態整合、入口索引與 handoff 可見性，而不是一次塞滿所有功能。

## Test Plan ID
- PBT-001

## Candidate Tool
- Notion

## 測試目標
- 是否能作為 ⓪ 公盤入口視圖
- 是否能承接最小狀態整合
- 是否能標示 handoff 與 next action
- 是否能與 GitHub 主本保持主寫 / 鏡像分工

## 最小測項
1. 建立一個 Seed 公盤頁面或表格
2. 放入至少三個入口項：治理核心、blockers、tool status
3. 標示來源為 GitHub 主本分支
4. 測試是否能清楚看到 next action 與 handoff target
5. 不進行主寫覆寫，只作入口與整合視圖

## 成功條件
- 可視為 ⓪ 公盤鏡像入口
- 不與 GitHub 主寫位衝突
- 能承接最小狀態整合與 handoff 顯示

## Notes
- 首輪實測通過後，才可討論是否升為公盤主工具
- 若 Notion 首測不理想，再切換 ClickUp 作替代測試
