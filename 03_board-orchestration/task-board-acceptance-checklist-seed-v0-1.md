# Task Board Acceptance Checklist — Seed v0.1

一句核心：
② 任務盤首輪不應只做出頁面或清單就算成功；需通過最小驗收檢查，才能視為可用的任務承接面。

## Checklist Items
- AC-01：Current Focus 與 Task Queue 可一眼看出目前任務重點
- AC-02：Responsible Board / Ref 清楚可見
- AC-03：Next Action 與 Status 可追讀
- AC-04：可從 ⓪ 公盤或 GitHub 入口 handoff 至任務盤
- AC-05：所有任務項都有 Source Ref
- AC-06：未出現反向主寫或覆寫 GitHub 的行為
- AC-07：更新流程明確為「先 GitHub，後任務盤」

## Result Options
- Pass：可作為任務盤首輪可用承接面
- Partial：可用但仍有欄位、責代或 handoff 對位不清
- Fail：不建議採用當前配置

## Notes
- 驗收結果應與 Task Board First Test Result Template 對齊
- 若 Partial / Fail，需回寫 blockers 與 routing seed
- 驗收通過不等於升為主工具，只代表首輪可用
