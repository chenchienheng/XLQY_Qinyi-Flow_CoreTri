# Task Board First Test Result Template

一句核心：
② 任務盤首輪實測完成後，應以固定結果模板記錄是否通過任務承接、責代可見性與 handoff 後續的最低要求。

## 最小欄位
- Test Result ID
- Candidate Tool
- Time
- Scope
- Passed Checks
- Failed Checks
- Writeback Position
- Related Board
- Next Action
- Notes

## Passed Checks 建議
- 任務盤可清楚顯示 Current Focus 與 Task Queue
- Responsible Board / Ref 可見
- Next Action 與 Status 可追讀
- 可從公盤入口 handoff 至任務盤
- 未發生主寫覆寫衝突

## 結果分類
- Pass：可作為 ② 任務盤首輪可用工具
- Partial：可用但需補欄位或調整 handoff
- Fail：不建議沿用該工具作任務盤首輪候選

## 原則
- 結果需可回讀，不得只憑主觀印象
- 若為 Partial / Fail，需標示補件或替代路徑
- 結果應回寫至 blockers、status record 與 routing seed
