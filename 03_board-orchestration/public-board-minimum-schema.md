# Public Board Minimum Schema

一句核心：
⓪ 公盤若要作為首輪入口鏡像，應先有最小頁面結構；沒有結構，公盤只會變成雜訊看板，不能承接狀態整合。

## 最小區塊
- Core Status
- Governance Core Entry
- Current Blockers
- Tool Family Status
- Next Action
- Handoff Target
- Source Ref

## 欄位建議
- Item ID
- Title
- Current Status
- Responsible Board
- Primary Write Ref
- Mirror Role
- Next Action
- Updated Time
- Notes

## 原則
- 公盤只承接入口與整合，不承接主寫覆寫
- 每個區塊都應可對回 GitHub 主本分支
- 未能對回來源者，不得列入正式公盤
- 公盤頁面應先少量、可讀、可 handoff
