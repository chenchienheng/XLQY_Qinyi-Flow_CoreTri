# Public Board First Test Result Template

一句核心：
⓪ 公盤首輪實測完成後，應以固定結果模板記錄是否通過入口鏡像、狀態整合與 handoff 可見性的最低要求。

## 最小欄位
- Test Result ID
- Candidate Tool
- Time
- Scope
- Passed Checks
- Failed Checks
- Writeback Position
- Related Board
- Next Action
- Notes

## Passed Checks 建議
- 公盤頁面可清楚顯示 Core Status
- Governance Core Entry 可對回 GitHub
- Current Blockers 與 Tool Status 可被整合展示
- Next Action 與 Handoff Target 可見
- 未發生主寫覆寫衝突

## 結果分類
- Pass：可作為 ⓪ 公盤首輪可用鏡像入口
- Partial：可用但需補欄位或調整結構
- Fail：不建議沿用該工具作公盤首輪候選

## 原則
- 結果需可回讀，不得只憑主觀印象
- 若為 Partial / Fail，需標示補件或替代路徑
- 結果應回寫至 blockers、status record 與 routing seed
