# Public Board Acceptance Checklist — Seed v0.1

一句核心：
⓪ 公盤首輪不應只做出頁面就算成功；需通過最小驗收檢查，才能視為可用的入口鏡像。

## Checklist Items
- AC-01：Core Status 可一眼看出 Seed / Stable / Main 狀態
- AC-02：Governance Core Entry 能對回 GitHub
- AC-03：Current Blockers 與 Tool Family Status 可被整合展示
- AC-04：Next Action 與 Handoff Target 清楚可見
- AC-05：所有入口項都有 Source Ref
- AC-06：未出現反向主寫或覆寫 GitHub 的行為
- AC-07：更新流程明確為「先 GitHub，後鏡像」

## Result Options
- Pass：可作為公盤首輪鏡像入口
- Partial：可讀但尚有缺欄或對位不清
- Fail：不建議採用當前配置

## Notes
- 驗收結果應與 Public Board First Test Result Template 對齊
- 若 Partial / Fail，需回寫 blockers 與 routing seed
- 驗收通過不等於升為主工具，只代表首輪可用
