# GitHub Native Board Acceptance Checklist — Seed v0.1

一句核心：
GitHub 原生盤不應只因為存在首頁就算成立；需通過最小驗收檢查，才能視為可用的靈絡原生盤面。

## Checklist Items
- AC-01：Current Layer Status 一眼可讀
- AC-02：Governance Core Entry 能對回有效 source refs
- AC-03：Current Blockers、Task Follow-up、Tool Status 可被整合展示
- AC-04：Log Entry Ref 與 Handoff Target 清楚可見
- AC-05：所有入口項都能對回 GitHub 主本文件
- AC-06：未出現第二主本或外部盤反向主導行為
- AC-07：更新流程明確為「先 GitHub 主本，後鏡像或輔助層」

## Result Options
- Pass：可作為靈絡原生盤入口
- Partial：可用但仍需補欄位或整理入口
- Fail：不建議視為正式原生盤入口

## Notes
- 驗收結果應回寫至 blockers、status map 與 governance index
- 驗收通過不等於 Stable，只代表盤面骨已可用
- 後續若外部盤接入，仍需服從此驗收基準
