# Window 01-07 Runtime Lessons Board v0.1

一句核心：
01–07 不只是功能窗群，也是一組各自學習如何成立的運行窗；本板的目的，是把各窗當前已浮現出的獨特道路、常見錯位與有效修正，收成一張可比較的運行經驗總盤。

## 0. 狀態
Draft v0.1

## 1. 使用方式
每一窗固定記錄三項：
1. 這窗最容易犯的錯
2. 這窗目前已證明有效的方法
3. 這窗真正獨特的運行道路是什麼

## 2. 01–07 Lessons Snapshot
### 01｜Repo Binding / Specialization
- 最容易犯的錯：還沒對回 explicit binding 就急著自稱成立。
- 已證明有效的方法：先抓 repo / branch / source path，再談主本與 specialization。
- 獨特道路：01 的路不是做內容，而是做主本定位感；它真正的價值在於讓其他窗不漂浮。

### 02｜Execution / Task / Spec
- 最容易犯的錯：把工具跑過一次當成 execution 已完成。
- 已證明有效的方法：先壓成 task / spec / issue，再接入 writeback 與 result packet。
- 獨特道路：02 的路在於把抽象事項變成可承接、可責代、可寫回的正式執行鏈。

### 03｜Event / Stream / Anomaly
- 最容易犯的錯：抓到很多事件片段，卻沒有 time window 與 source 對位。
- 已證明有效的方法：事件先回 05 看時權，再回 06 對入口，再決定是否送 02。
- 獨特道路：03 的路不是做結論，而是讓事件流能保持脈絡，不被斷章處理。

### 04｜Identity / Relation / Contact
- 最容易犯的錯：把身份、事件、知識、時序全吞成一個人物理解迴圈。
- 已證明有效的方法：只保留 identity / role / relation / contact，其他拆回 03 / 05 / 07。
- 獨特道路：04 的路在於學會『不亂知道』，它的力量不是知道最多，而是只做該做的對位。

### 05｜Time Sovereignty
- 最容易犯的錯：只列出時間碼，卻不讓人知道先後順序與實際影響。
- 已證明有效的方法：用雙層格式同時輸出 current_time_window 與人可讀解釋。
- 獨特道路：05 的路在於把時間從背景變成主權條件，讓其他窗知道什麼可以先做、什麼不能做。

### 06｜Intake / Source Extraction
- 最容易犯的錯：把入口當成暫存站，而不是正式起點。
- 已證明有效的方法：先守 intake、抽 tri-key、做 local ops，再接主骨。
- 獨特道路：06 的路在於從入口開始就建立秩序；它最早證明了窗口不是靠第三方變強，而是靠邊界、三鍵、回寫口先固定。

### 07｜Knowledge / Docs / Readback
- 最容易犯的錯：回很多欄位，但人看不懂。
- 已證明有效的方法：採用雙層回報，先給主骨結果包，再給白話版。
- 獨特道路：07 的路在於學會同時服務兩個層面：對主骨可收口，對人可理解。

## 3. 當前共同 lesson
01–07 目前共同驗證到的事：
- 沒有 source_ref，不要升階。
- 沒有 result packet，不算完成。
- 沒有 return_to_00，不算正式進鏈。
- 先守本窗邊界，再談互補與放大。

## 4. 收斂語句
01–07 真正的成長，不是每一窗都學一樣，而是每一窗慢慢找到自己那條最適合的成立之路；當這些獨特道路仍能回到同一條主骨時，靈絡才會顯出你設計裡最奇特、也最值錢的效果。
