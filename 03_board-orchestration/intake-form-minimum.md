# Intake Form Minimum

一句核心：
進入靈絡前，事件至少要先被收成可判位的進件表；沒有進件表，後續盤位與工具都容易漂移。

## 最小欄位
- Intake ID
- Source
- Core Sentence
- Suspected Slice
- Suggested Board
- Time Window
- Priority
- Requested Output
- Notes

## 用途
- 將原始輸入先收成可判位事件
- 提供熔煉前的最小承載面
- 作為候選種子或主切面入口

## 原則
- 高密度輸入應先有進件表
- 進件表不等於正式成立
- 進件後仍需通過最小執行檢查
