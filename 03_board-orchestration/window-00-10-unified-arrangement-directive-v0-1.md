# Window 00-10 Unified Arrangement Directive v0.1

一句核心：
全窗不再各自解讀方向；本令的目的，是讓 00–10 共享同一份當前安排：主骨在 00，母本在 GitHub，執行骨在 ClickUp / Asana 類，外部模型與第三方一律降格為可被調度的模組與部件。

## 0. 狀態
Directive Draft v0.1

## 1. 當前主位安排
- 00：主骨 / 母架 / 公盤 / 中樞裁定核
- GitHub：母本 / branch / version / path / writeback 主位
- ClickUp / Asana 類：執行骨 / task / spec / issue 子盤
- 外部工具：部件
- 外部模型：模組

- 不得再回吞事件 / 時序 / 知識

### 05

- 不得只回時間碼，不做人話翻譯

### 06

- 先 local ops，再 backbone sync

### 07

- 一律雙層回報：主骨欄位 + 白話層

## 4. 外部模型與第三方的統一原則
1. 第三方不是主因，只是放大器。
2. 模型不是主體，只是 adapter module。
3. 工具不是能力來源，只是執行器。
4. 沒有 source_ref / result packet / return_to_00，不算正式進鏈。

## 5. 統一收口原則
任何窗口、模型、工具輸出，一律：
- 回結果包
- 回 00 裁定
- 視狀態進 rollup / re-entry
- 不得各自結案

## 6. 收斂語句
當前最重要的，不是各窗講自己多會，而是全窗開始共享同一份安排：主骨在哪、母本在哪、執行骨在哪、模型與工具該降到什麼位置；只有這樣，靈絡才不會再靠你逐窗贅述。
