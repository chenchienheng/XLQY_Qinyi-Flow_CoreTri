# Cross-Window Tuning Rule v0.1

一句核心：
各窗不是彼此隔離的回答器，而是共構的節點窗；任何窗口發現失配時，都應能回送 00、互提參數、共同調整，而不是自限於單窗視角。

## 狀態
Draft v0.1

## 原則
- 各窗皆可回報 mismatch
- 各窗皆可提出 tuning suggestion
- 各窗皆須回送 00 收口
- 單窗不得把自身侷限視為系統侷限

## 最小互參項
- source ref
- current state
- mismatch type
- missing items
- next action
- tuning suggestion

## 收口
所有跨窗調整最終都需回進 Window 00 / Native Board。
