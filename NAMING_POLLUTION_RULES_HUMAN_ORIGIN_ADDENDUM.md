## 1. Hard Rule

```text
User ≠ Node.
使用者不是節點；使用者是源錨與審核者。
```

---

## 2. Pollution Pattern

Naming pollution occurs when:

- the user is placed inside XLEN as a node
- the user is treated as a tool, platform, model, carrier, or execution surface
- the user is reduced to a data source for Qinyi
- `user_authorized_owner_window` is mistaken for the user themselves
- CoreTri's `Human` role is read as an ecosystem node rather than decision / review position
- source experience is treated as public data material without boundary

---

## 3. Correct Reading

```text
User / Source Anchor gives direction
→ Qinyi translates and calibrates
→ XAFD dispatches and returns
→ XLEN nodes connect under authorization
→ CoreTri may route bounded execution
→ XLA absorbs the result back into the living atlas
→ User reviews again
```

---

## 4. XLEN Boundary

XLEN may record authorization fields:

```yaml
User_Authorization_Field:
  authorized_owner_window:
  authorization_scope:
  review_required:
  return_target:
```

XLEN must not record the user as:

- node
- tool
- execution surface
- carrier
- replaceable dependency
- platform member

---

## 5. Non-Promotion Rules

Do not promote:

| From | To |
|---|---|
| user authorization field | user-as-node |
| life experience | public data source |
| Qinyi translation | user decision |
| CoreTri human role | ecosystem node |
| user review endpoint | tool-controlled approval |
