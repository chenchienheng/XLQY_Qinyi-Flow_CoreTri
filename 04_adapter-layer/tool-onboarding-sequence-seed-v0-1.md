# Tool Onboarding Sequence — Seed v0.1

一句核心：
在 GitHub 主骨已立的前提下，其他工具家族不應同時全開；應依主鏈需求與阻塞項，按順序逐步接入。

## 建議順序
1. GitHub（已測）
2. Gamma（已測）
3. ⓪ 公盤候選工具（Notion 或 ClickUp）
4. ② 任務盤候選工具（Asana 或 ClickUp）
5. ③ 資產盤候選工具（Drive）
6. ⑤ 通訊盤候選工具（Gmail / Slack / Calendar / Contacts）
7. ⑦ 窗口盤與展示鏡像

## 判定依據
- 是否能補當前 blocker
- 是否已有明確主寫或鏡像定位
- 是否已有 writeback target
- 是否能以低負載完成首輪實測

## 原則
- 先補主鏈阻塞，再擴大家族覆蓋
- 未通過首輪實測者，不得列為 stable 工具
- 每完成一輪接入，應更新 status map、routing seed 與 blockers
