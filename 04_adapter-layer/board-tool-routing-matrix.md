# Board Tool Routing Matrix

一句核心：
盤位先決定作用，再選工具；工具不得反向主導盤位。

## 最小對位
- ⓪ 公盤：狀態整合 / handoff / writeback index
- ① 主案盤：核心文本 / 決策 / version / freeze
- ② 任務盤：排程 / 行動項 / 責代追蹤
- ③ 資產盤：文件承載 / 鏡像 / 回讀來源
- ④ 圖像盤：視覺原型 / 海報 / 簡報
- ⑤ 通訊盤：mail / slack / contacts / calendar
- ⑥ 程序盤：GitHub / schema / rule / adapter
- ⑦ 窗口盤：window naming / 對外界面 / 展示版本
- ⑧ 日誌盤：log / snapshot / 歷史追讀

## 工具優先建議
- 公盤：Notion / ClickUp / Log Index
- 主案盤：GitHub / Notion
- 任務盤：Asana / ClickUp / Calendar
- 資產盤：Drive / GitHub Ref / Mirror Layer
- 圖像盤：Gamma / Adobe Family / Slides
- 通訊盤：Gmail / Slack / Contacts / Calendar
- 程序盤：GitHub
- 窗口盤：Gamma / Docs / Shared View
- 日誌盤：GitHub / Drive / Log Schema

## 原則
- 先判盤，再選工具
- 同盤可多工具，但需有主寫與鏡像之分
- 工具切換需保留 writeback target
- 無盤位歸屬者，不得視為正式成立
