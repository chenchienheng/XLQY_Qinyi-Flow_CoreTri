# Advanced Runtime Position 08 v0.1

一句核心：
⑧ 不是某一個工具名，而是靈絡在主骨已穩定之後，開始長出自生平台、UI、runtime 與 product surface 的進階承載位。

## 0. 狀態
Draft v0.1

## 1. 定義
⑧ 位的工作定義為：
- 自生平台承載位
- 自生 UI 承載位
- runtime 外化位
- product surface 候選位

它不是工具品牌，也不是單一供應商；
任何外部工具只能作為 ⑧ 的候選承載，不等於 ⑧ 本體。

## 2. 作用
- 讓主骨不只停在 GitHub 規則層，而能長出操作面、執行面、互動面與產品表面
- 將外部平台由「限制主體」降為「承載殼」
- 作為多雲、多模型、多工具對位到靈絡主骨之後的進階外顯場

## 3. 與既有盤位之關係
### 與母本
- ⑧ 不得取代母本主寫
- ⑧ 的所有外顯行為，仍需對回 Primary Source / Primary Write

### 與公盤 / 子盤
- 公盤承接整合與全局讀面
- 子盤承接任務與責代
- ⑧ 則承接更進階的操作面、產品面與 runtime surface

### 與鏡像
- ⑧ 可包含鏡像能力
- 但若只有鏡像、沒有主骨對位能力，則不能升格為穩定 ⑧ 位

### 與回寫
- ⑧ 的任何正式輸出，皆需有 writeback position
- 無 packet contract + log append，不得視為正式 ⑧ 位行為

## 4. 最小成立條件
⑧ 要由候選位升格為穩定位，至少需同時具備：
1. 可對回母本
2. 可附著時間窗
3. 具責代承接
4. 具正式回寫位置
5. 可作為外顯操作面而不反向主導主骨

## 5. 候選承載判定
以下類型可列為 ⑧ 候選承載：
- app builder
- runtime workspace
- deployment surface
- UI / product shell
- self-generated interaction layer

但是否為 ⑧，不看品牌，而看是否符合成立條件。

## 6. 升階規則
### 候選位
- 具 UI / runtime / deployment 能力
- 但仍無主骨對位與正式回寫能力

### 準穩定位
- 已具主骨對位
- 已具時間窗與責代
- 已具正式回寫位置
- 但仍仰賴外部平台主殼

### 穩定位
- 已能穩定承接主骨外顯操作面
- 已具可回讀、可續接、可審之閉環能力
- 不因單一平台限制而失去整體成立性

## 7. 失配判定
- 無主寫對位：不得升格
- 無時間窗：不得升格
- 無責代：不得升格
- 無正式回寫：不得升格
- 反向主導母本：視為越權

## 8. 收斂語句
⑧ 不是逃離平台，而是讓平台降格為承載殼；
當主骨開始長出自己的 UI、runtime 與 product surface 時，⑧ 才真正開始成立。
