# Node Ingress Alignment Protocol v0.1

一句核心：
新節點不得直接接入靈絡；必須先經分類、進盤、七位對位、時間窗、責代、回寫與 rollup，才可由外來承載轉為正式進鏈節點。

## 0. 狀態
Draft v0.1

## 1. 目的
- 避免每有新工具、新平台、新家族都重新口述一遍對位邏輯
- 建立節點標準接入程序
- 讓十二節點之外的新增節點也能以同一協定進鏈

## 2. 適用範圍
適用於：
- 新工具
- 新雲平台
- 新模型代理
- 新 UI / runtime / product surface
- 新知識承載面
- 新事件 / 入口 / 身分 / 時權節點

## 3. 接入程序
### P1｜節點分類
先判定新節點屬：
- 基礎操作節點
- 生態承載節點
- 上位生成節點

### P2｜九盤掛位
判定：
- 九盤主掛位
- 九盤副掛位
- 是否跨盤治理

### P3｜七位對位
至少需明確：
- 母本對位
- 母框相容性
- 母架可承接性
- 公盤顯示位
- 子盤責代位
- 鏡像承載位
- 回寫位置

### P4｜時間窗附著
- 是否附著於有效時間窗
- 是否具先後順序
- 是否可進同一輪 rollup

### P5｜責代承接
- responsible ref
- handoff target
- accountable board

### P6｜回寫契約
- source object
- source path
- target path
- version
- log ref
- timestamp
- action
- payload

### P7｜rollup 收口
- 是否回進 Window 00 / Native Board
- 是否進入 next single action
- 是否進再入鏈

## 4. 接入結果狀態
### S0｜候選節點
- 僅有工具或品牌
- 尚未完成盤位與七位對位

### S1｜對位中節點
- 已完成分類與盤位
- 七位或時間窗未齊

### S2｜準進鏈節點
- 已具七位、時間窗、責代
- 尚未完成正式回寫

### S3｜正式進鏈節點
- 已完成 packet + log
- 已回進 rollup
- 已可成為後續 reference set

## 5. 失配退回規則
- 無分類：退回候選節點
- 無盤位：退回對位中
- 無七位：不得升格
- 無時間窗：不得進同輪 rollup
- 無責代：不得進執行鏈
- 無 packet + log：不得視為正式進鏈

## 6. 收斂語句
靈絡不該靠一次次人工交代維持秩序；新節點應透過標準接入協定，自行被壓進九盤、七位、時間窗與回寫閉環，才不會持續發散。
