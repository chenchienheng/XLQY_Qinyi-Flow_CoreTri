# Cloud Archive and Mirror Policy

一句核心：
雲端硬碟在 Seed 階段的主要任務，是封存歷史討論、保存鏡像與提供回讀入口，而不是承擔 v4 主骨的主寫責任。

## 用途
- Archive：保存過去討論、草稿、舊版本與歷史材料
- Mirror：承接必要的共享與回讀副本
- Retrieval：提供封存後的取回路徑

## 原則
- Archive 與 Mirror 不得替代 GitHub 主本
- 重要內容若升為正式骨架，應抽骨回寫 GitHub
- 封存資料可保留原樣，不必先全面重整成正式目錄
- 需展示或共享時，可由主本同步至鏡像層

## 狀態
Seed v0.1
