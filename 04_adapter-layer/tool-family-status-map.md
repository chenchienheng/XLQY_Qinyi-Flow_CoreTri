# Tool Family Status Map

一句核心：
工具家族不只列名稱，還必須標示其在靈絡中的角色、負載級別、當前狀態與回寫位置。

## 最小欄位
- Tool Family
- Role Type
- Load Level
- Primary Board
- Writeback Target
- Status
- Notes

## 初始分類
- GitHub：Procedure / Core / ⑥ 程序盤 / version + rule
- Notion：Board / Support / ⓪ or ① / board state
- ClickUp：Task / Support / ② 任務盤 / action tracking
- Asana：Task / Support / ② 任務盤 / action tracking
- Drive：Asset / Mirror / ③ 資產盤 / mirror + source readback
- Gamma：Visual / Burst / ④ 圖像盤 / prototype view
- Gmail：Communication / Support / ⑤ 通訊盤 / thread record
- Slack：Communication / Support / ⑤ 通訊盤 / conversation record
- Calendar：Schedule / Support / ⑤ 通訊盤 / time window record
- Contacts：Identity / Support / ⑤ 通訊盤 / contact reference

## 原則
- 狀態需區分：planned / connected / tested / stable / paused
- 每次工具實測後應更新 status 與 notes
- 無 writeback target 的工具只可暫列，不得視為穩定接面
