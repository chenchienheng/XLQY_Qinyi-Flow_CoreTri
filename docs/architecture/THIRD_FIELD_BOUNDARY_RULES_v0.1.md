# Third Field Boundary Rules

To ensure the Third Field functions strictly as an intermediary Field Semantic
Dependency Network and protects the invariant core, the following boundary rules
must be strictly enforced:

- 外部需求不得直接改寫中心核。
- 外部訊號必須先經過場域辨識。
- 未證實資訊不得進正式結論。
- 私人真人來源不得外顯。
- 工具只是球面節點，不得定義任務本質。
- 不同窗口不得混核。
- 不同生態不得互相污染。
- 所有輸出必須能回流母架、窗口或任務鏈。

## Carrying Capacity and Load Boundary

A connection being possible does not mean the connected chain can safely carry
every load. The Third Field must enforce load boundary limits during mapping:

1. **Carrying Capacity Check**: Each chain, window, or node has a specific
   load-bearing range. The Third Field must verify if the destination window
   can safely carry the task weight.
2. **Analogy of Force**: A thread can sew fabric, but a thread should not be
   treated as a rope. Mappings must respect the structural strength of the
   assigned node.
3. **Overload and Mismatch Routing**: If a task weight exceeds a chain's
   carrying capacity, the Third Field must not force a weak chain to carry it.
   It must route the mismatch to a review surface or request structural scaling
   before proceeding.
