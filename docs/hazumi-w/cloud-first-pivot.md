# HAZUMI.W Cloud-first Pivot v0.2

Status: Candidate / Cloud-first / Human-gated / No Runtime
Master Gate: #297

## Core

HAZUMI.W is moving from local experiment framing to cloud-first documentation and review framing. The active carrier is GitHub / DCP as a reviewable cloud substrate. The company-side build remains manual and permissioned by Vitas inside allowed company surfaces.

## What Changed

- Qinyi is the front-facing GPT cloud review bridge.
- Hazumi / Codex is the background cloud docs, schema, and manual-guide support lane.
- Jules is an optional branch/code/test candidate lane only after explicit approval.
- GitHub / DCP carries issues, comments, docs, schemas, and return packets.
- #297 is the current master gate.

## What Did Not Change

- Candidate is not Approved.
- BuildReady is not Runtime.
- Manual guide is not completed company build.
- GitHub issue/comment is not installed agent.
- GitHub docs are not deployed system.

## Cloud-first Rules

- Read, plan, and draft before write.
- Inventory before patch.
- Return Packet before closeout.
- Semantic Firewall before automation.
- Human gate before merge, publish, or company-side build.

## Legacy Treatment

Local experiment materials are valuable as source library and historical nutrient. They may be used to extract root grammar, state models, return formats, semantic firewall, manual build boundary, and what-not-to-claim material. They should not be treated as active runtime packages.

## Final Judgment

Cloud-first does not mean autonomous execution. It means reviewable cloud substrate, bounded support lanes, and human-gated action.
