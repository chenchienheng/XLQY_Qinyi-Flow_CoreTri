# G vs M Carrier Comparison Candidate

Status: Candidate / Domain Pack Support / No Runtime

## Core

G and M are both carrier ecologies. Neither is the governance core.

## Comparison

```yaml
G_Ecosystem:
  nature: "personal integrated carrier chain"
  examples:
    - "Gmail signal"
    - "Drive carrier"
    - "Calendar time gate"
    - "Contacts index"
    - "Mirror review"
  authority: "personal authority"

M_Ecosystem:
  nature: "multi-mode Microsoft carrier projection"
  modes:
    - "Personal_M"
    - "Company_M365"
    - "Unknown_M"
  authority: "mode-dependent"
```

## Shared Rule

Pattern is portable. Data is not. Carrier can change. Authority must be re-established.

## Red Doors

- Personal_M != Company_M365.
- G maturity != M approval.
- Shared syntax != shared data.
- Carrier migration != authority migration.
