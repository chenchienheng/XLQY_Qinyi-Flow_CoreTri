## Core

This example shows how the G ecosystem can be treated as one carrier governance chain rather than scattered tools.

```text
Gmail Signal → Qinyi Judgment → Drive Carrier → Mirror Review → Return Packet → Vitas Decision → Archive / Upgrade / Schedule
```

## Tool Positions

- Gmail is a signal gate.
- Drive is a carrier and state layer.
- Mirror review is a second-opinion surface, not approval.
- Calendar is a time gate.
- Contacts is a human / organization index.
- Qinyi frames judgment and return.
- Vitas decides.

## Red Doors

- Gmail Label != Responsibility.
- Drive File != Closeout.
- Mirror Review != Approval.
- Calendar Event != Task Completion.
- Contacts Entry != Authorization.
- Summary != Decision.
- Record != Execution.
- Schedule != Follow-through.

## Evidence Boundary

If Qinyi did not directly verify a Drive file, folder, or setting, mark it as user-reported material.

## Placement

This is a domain example. It is not Open Core itself, not a Google automation claim, and not a runtime deployment.
