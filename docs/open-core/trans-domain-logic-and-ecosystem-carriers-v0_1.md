## Core

XuanLing should not be treated as an add-on to G ecosystem, M ecosystem, family systems, brand systems, or any specific platform. These ecosystems are carriers or domain expressions. XuanLing is the shared governance grammar that can pass through them.

This is trans-domain logic, not merely cross-domain integration.

## Cross-Domain vs Trans-Domain

```yaml
Cross_Domain:
  meaning: "connect one domain to another"
  risk:
    - "each domain keeps its own logic"
    - "bridges multiply"
    - "authority and state may diverge"

Trans_Domain:
  meaning: "the same bottom governance grammar re-instantiates across different ecosystems"
  core:
    - "Source"
    - "Carrier"
    - "Authority"
    - "Gate"
    - "Action"
    - "Return"
    - "Rebuild"
```

In this model, the goal is not to connect many unrelated tools. The goal is to let one governance grammar remain consistent across many carriers.

## M Ecosystem Clarification

M ecosystem is not only company M365. It has at least two modes:

```yaml
M_Ecosystem_Modes:
  Personal_M:
    nature: "personal Microsoft cloud carrier"
    possible_surfaces:
      - "Outlook"
      - "OneDrive"
      - "Calendar"
      - "To Do"
      - "OneNote / Loop if available"
    authority: "personal authority"
  Company_M:
    nature: "enterprise permissioned operational carrier"
    possible_surfaces:
      - "Outlook"
      - "Teams"
      - "SharePoint"
      - "Lists"
      - "Power Apps"
      - "Power Automate"
    authority: "company / tenant / owner / admin / human gate"
```

No computer endpoint is required for the conceptual chain. Endpoint access is only one carrier. Cloud-side M surfaces can still be governed through docs, categories, manual guides, and return packets.

## Shared Bottom Platform

The shared bottom platform is not G or M. It is the governance chain:

```text
Source → Carrier → Authority → Gate → Action → Return → Rebuild
```

Every ecosystem can instantiate this chain:

```yaml
Instantiation:
  G:
    source: "Gmail / Drive / Calendar / manual signal"
    carrier: "Drive / GitHub-DCP / Calendar"
    authority: "Vitas"
    return: "Qinyi review / Vitas decision"
  M:
    source: "Outlook / Teams / SharePoint / manual report"
    carrier: "OneDrive / SharePoint / Lists / document"
    authority: "personal or company authority depending on mode"
    return: "Qinyi review / Vitas or company decision"
```

## Red Doors

- Ecosystem maturity != XuanLing completion.
- G grammar stabilized != M implementation approval.
- Personal M authority != Company M authority.
- Cloud carrier != endpoint permission.
- Shared syntax != shared data.
- Domain pack != Open Core.
- XuanLing support claim != public doctrine.

## Final Rule

XuanLing should be understood as the common governance base. Ecosystems are expressions of that base, not the source of the base. A domain pack may map the base into G, M, family, brand, education, engineering, or public contexts, but none of those domains should capture or define XuanLing itself.
