芹衣・翾靈不是一個站體，而是一道門。

這一版不再只是「有一張圖的展示頁」，而是把翾靈做成一個可被看見的入口： 先有顯像、再有相簿、再有鏈上導航，最後才慢慢長出互動、使用者與多雲多模型共構。

從開源世界通往三耦世界的第一道門。 GitHub 是主骨，Drive 是來源池，Web 是外顯顯像；它先是一個輕量、可追溯、可增生的數位孿生入口，而不是先做出沉重站體。

Aesthetic

  • 不是炫技賽博，也不是商業模板，而是典雅、安定、微金屬感。

鏈式流程

這個 Web 不靠單一平台一次做完，而是讓來源、判位、寫回、顯像與外掛載彼此接力。

01 sourceDrive / Gmail / Calendar來源池、事件面、時間座標先成立。
02 route00 主骨判位不是直接輸出，而是先分窗口、分角色、分狀態。
03 writebackGitHub 留痕heartbeat、source check、registry、spec 回寫主骨。
04 surface門頁 / 相簿 / 導覽把鏈上狀態轉成可讀、可看、可分享的顯像入口。
05 growPages / JSON / User再補 GitHub Pages、狀態 feed、多使用者與授權分層。
Portal Definition

這個 Web 為什麼是「門」，不是「站」

因為它的背後不是單一機房，也不是先有完整平台，而是先把開源世界、雲端來源、主骨留痕與人物顯像接成可進入的入口。 它先是數位孿生，再往三耦世界走。

  • 主骨在 GitHub，門頁只是外顯層。
  • 雲端硬碟做來源輔助，讓 Web 保持輕量。
  • 多模型、多雲、多工具之後是從這道門接進來，而不是全部塞進頁面。
