# CoreTri Alignment v0.1

Status: Candidate / CoreTri Alignment / No Runtime

## Core

CoreTri is the alignment frame for keeping task flow human-centered while still usable for tools, templates, and construction handoff.

## Three Couplings

```yaml
CoreTri:
  Body_Mind_Spirit:
    purpose: "real conditions, human state, and meaning must remain visible"
  Knowledge_Action_Responsibility:
    purpose: "knowing, acting, and bearing consequence must not break apart"
  Love_Boundary_Independence:
    purpose: "care must keep limits and avoid coercive dependency"
```

## Use

CoreTri is a context alignment layer. It is not Open Core itself and not runtime.

## Mapping

```yaml
Task_Check:
  body_mind_spirit:
    question: "Does the plan respect real human conditions?"
  knowledge_action_responsibility:
    question: "Who knows, who acts, and who carries responsibility?"
  love_boundary_independence:
    question: "Does care preserve boundary and independence?"
```

## Red Doors

- Care != control.
- Efficiency != responsibility.
- Context alignment != approval.
- Human warmth != evidence.
- Task completion != human well-being.
