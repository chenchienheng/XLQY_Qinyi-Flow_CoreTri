## Supersession Note

This v0.1 file used the phrase "integration windows," which can misread the design as separate ecosystem windows.

The corrected model is:

```text
One integration window, multiple phases, multiple streams.
```

Gmail is not a separate window from Drive, Calendar, Contacts, Gemini, or GitHub/DCP. Gmail is the first completed or crossed phase-stream inside the same G ecosystem integration window. The current work has moved past Gmail into other G ecosystem streams.

Use the v0.2 phase-stream contract instead:

```text
docs/settings/g-ecosystem-phase-stream-contract-v0_2.md
```

## Historical Core

The original intent remains useful: G ecosystem and M ecosystem may share governance syntax, but must not share data, authority, or responsibility.

However, the v0.1 wording should not be used as the active contract because it may imply separate windows instead of one window moving through phases.
