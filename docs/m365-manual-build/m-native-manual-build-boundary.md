# M-native Manual Build Boundary｜Candidate

Status: Candidate / Manual Build Guide Boundary / No Runtime
Master Gate: #297

## Core

M-native means carrier alignment, not authorization granted. Manual build guide means Vitas can build manually inside allowed company surfaces; it does not mean the company system has already been built.

## Allowed AI Outputs

- Requirements analysis.
- SharePoint List field schema.
- Power Apps screen specification.
- Flow trigger/action design.
- Permission model draft.
- Manual build checklist.
- Mock data test cases.
- Return packet form.

## Not Claimed

- No company tenant operation by AI.
- No completed company build.
- No runtime deployment.
- No approval proof.
- No public release.

## Company Minimum Version

The company-facing version should remain a Shared Work Flow:

- Demand intake.
- Work order.
- Responsible person.
- Status.
- Due date.
- Review.
- Return / revision.
- Completion record.

This is a practical minimum work chain. It does not expose the full XuanLing world model, three-coupling structure, internal Qinyi/Hazumi/Jules contracts, or platform anti-capture doctrine.
