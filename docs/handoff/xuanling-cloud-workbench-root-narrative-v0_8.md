# 翾靈雲端主幹報告 v0.8｜時間主權下的權圈責代生態

Status: Candidate / Root Narrative / Not Approved / No Runtime / No External Writeback Without Vitas Approval
Master Gate: #297 DCP-GITHUB-CLEANUP-v0.7.2

## 一句核心

翾靈雲端主幹不是工具整合，而是時間主權下的權圈責代生態：每個任務、文件、判斷與行動，都必須保留其出生時間、承載位置、權限狀態、證據、回流位置與可重建方式；下一階段不能偷走上一階段尚未完成的審查與責任。

## 1. Positioning

本文件將 v0.7 主文升格為 XuanLing Cloud Workbench root narrative 候選。它不是施工包、不是 checklist、不是公司政策、不是 M365 授權、不是 GitHub merge approval，也不是 runtime 宣稱。

它的作用是把 DCP、DXC、XDDC、XLEN、XAFD、XLQY、CoreTri、Qinyi、Hazumi 到 XuanLing 的演化，收束成雲端可審查的主幹語言。

## 2. Capability Expansion vs Capability Governance

大公司多從 capability expansion 切入：算力、模型、平台、產品、資料中心、開發者生態與市場控制。翾靈從 capability governance 切入：能力進入世界後，誰負責、權限是否偷渡、狀態是否升格污染、平台是否俘虜語法與正本、行動後如何回流、修正與重建。

因此，本系統不以 API、MCP、OAuth 或 runtime 為第一入口，而以 Source / Carrier / Authority / Gate / Action / Return / Rebuild 為第一入口。

## 3. Time Sovereignty

時間主權定義：每個任務、文件、判斷、行動與回流，都必須保留它在時間中的狀態、權限、證據與責任。下一個狀態不能偷走上一個狀態尚未完成的審查與承擔。

Core rules:

- Draft 不能偷渡成 Candidate。
- Candidate 不能偷渡成 Approved。
- Approved 不能偷渡成 Runtime。
- Return Packet 不能偷渡成 Closeout。
- Review 不能偷渡成 Human Approval。
- Capability 不能偷渡成 Permission。

最準一句：權圈不是靜態階級，而是時間中的責任圈。

## 4. Authority Rings in Time

- R0 Human Source: 最終批准與責任承接。
- R1 Governance Trunk: 定義狀態、門、語意防火牆。
- R2 Delivery Trunk: 把候選材料整理成可審查交付。
- R3 Candidate Branch: 支撐、試作、推導。
- R4 Evidence QA: 證據、manifest、crosswalk、diff。
- R5 Temp Debug: 暫存、排錯、渲染。
- R6 External Relay: GitHub / DCP / public-safe / human relay。

## 5. Cloud Workbench Turn

本地實驗的意義不是證明 local-only 安全，也不是證明 endpoint、PowerShell、registry、services 可任意使用；相反，它暴露了 carrier 貼近 endpoint 時容易被誤讀與污染。因此本階段轉向雲端可審查底盤：

- 先 inventory，不先施工。
- 先 Return Packet，不先 runtime。
- 先 Semantic Firewall，不先 API / OAuth / MCP。
- 先 human-gated build，不先自動化。

## 6. GitHub / DCP Role

#297 是 Cloud Workbench Master Gate 候選。它不是單一 issue，而是 cleanup / inventory / review / return packet 的主入口。GitHub 的首要任務不是 merge 所有東西，而是建立 Repo State Map。

Repo State Map:

- Keep: 保留為有效節點。
- Merge: 可合併，但需明確 Vitas approval。
- Supersede: 被新主幹覆蓋，保留歷史。
- Park: 有價值但不進 active lane。
- Red Gate: 不得執行，只保留停止理由。
- Needs Vitas Decision: 需要使用者裁定，不可由 Codex / Qinyi 自決。

## 7. Company Minimum Version

公司端只保留最小共用作業底盤：以 Microsoft Lists、Power Apps、Power Automate、SharePoint、Teams 為候選 carrier，整理需求入口、工作單、責任人、狀態、期限、審查、退回與完成紀錄。AI 只協助整理欄位、流程與手動建置指引，不碰公司資料、不自動操作系統。

完整翾靈世界觀不需送入公司端；公司端只需要可審核、可回流、可落地的最小工作鏈。
