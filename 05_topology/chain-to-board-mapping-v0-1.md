# Chain-to-Board Mapping v0.1

一句核心：
鏈簇若不對位到九盤、七位與窗群，仍只是分類知識；本文件的目的，是將不同鏈型正式壓回靈絡的可運行治理座標。

## 0. 狀態
Draft v0.1

## 1. 使用目的
- 將鏈簇分類由概念層壓成盤位層
- 讓不同鏈能直接對位九盤、七位與主要窗口
- 作為後續節點接入、審查、回寫與 rollup 的映射基底

## 2. 映射原則
每一條鏈至少對位以下四項：
1. 九盤主掛位
2. 九盤副掛位
3. 七位主對位
4. 主要窗口

## 4. 映射判讀規則
### 4.1 單主掛位，雙副掛位優先
除依存鏈外，大多數鏈型應有一個主盤位與一到兩個副盤位，避免全盤泛掛導致失焦。

### 4.2 七位決定成立層級
- 對位母本：偏來源與版本主權
- 對位母框：偏成立邊界與一致性
- 對位母架：偏運行脊椎與續接能力
- 對位公盤：偏入口整合與全局讀面
- 對位子盤：偏責代、執行與承接
- 對位鏡像：偏外顯承載與回讀
- 對位回寫：偏 packet、log 與閉環證據

### 4.3 窗口決定實際處理位置
- 01 偏 repo binding / specialization
- 02 偏 execution / spec / issue / process
- 03 偏 event / stream / anomaly
- 04 偏 identity / relation / contact indexing
- 05 偏 time sovereignty / effective order
- 06 偏 intake / source extraction
- 07 偏 knowledge / docs / readback
- 08 偏 advanced runtime / UI / product surface

## 5. 使用方式
### 用法 A｜由鏈找盤
先判斷當前任務更接近哪一類鏈，再定位其主掛位、副掛位與主要窗口。

### 用法 B｜由盤找鏈
若任務先落在某盤位，再反查其背後可能屬於哪一種鏈型，以免盤位使用漂移。

### 用法 C｜由失配找鏈
當某窗口或盤位出現失配時，先判斷是：
- 承接鏈失配
- 時序鏈失配
- 回寫鏈失配
- 重建鏈失配
而非直接以單窗失敗描述整體問題。

## 6. 收斂語句
鏈簇的分類，不是為了讓世界更複雜，而是為了讓不同曲面、不同節點、不同盤位與不同窗口，都能被壓進同一套可對位、可回寫、可重建的治理地圖。
