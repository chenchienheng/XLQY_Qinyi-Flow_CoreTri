# Chain Evidence Maturity Matrix v0.1

一句核心：
鏈不是被命名後就成熟；是否成熟，取決於它是否已有可證鏈、可承接鏈、可回寫鏈與可重建鏈。本矩陣的目的，是為各類鏈提供共同成熟度判準。

## 0. 狀態
Draft v0.1

## 1. 四種核心證據
### E1｜可證鏈
- 是否可對回 source / version / path / window
- 是否可被七判審查

### E2｜可承接鏈
- 是否可進盤
- 是否有 responsible ref
- 是否有 next action

### E3｜可回寫鏈
- 是否有 packet contract
- 是否有 target path
- 是否有 log append
- 是否可回送 00

### E4｜可重建鏈
- 是否有 recovery point
- 是否有 snapshot
- 是否有 audit trail
- 是否可再入鏈

## 2. 成熟度等級
### M0｜命名層
- 只有鏈名或概念
- 尚無 source / path / process

### M1｜描述層
- 已能說明最小成立條件
- 但尚未進盤、進窗或進七位

### M2｜對位層
- 已能對位九盤、七位與主要窗口
- 但回寫與重建能力未成型

### M3｜運行層
- 已具可證鏈與可承接鏈
- 可在窗口間形成結果包與回送 00

### M4｜閉環層
- 已具 packet、log、rollup、re-entry
- 可視為正式閉環運行鏈

### M5｜生態層
- 已具重建能力
- 已可在多盤、多窗、多節點中穩定再入鏈
- 可視為成熟鏈生態單元

## 3. 各鏈初始成熟度建議
| 鏈型 | 初始成熟度 | 原因 |
|---|---|---|
| 區塊鏈 | M3 | 可證鏈與承接鏈普遍較強，但未必接入靈絡閉環 |
| 供應鏈 | M2 | 常具流程與責代，但未必具正式回寫與再入鏈 |
| 價值鏈 | M1 | 多停留於商業敘事，缺 formal evidence chain |
| 食物鏈 | M1 | 屬自然依存鏈，非典型治理閉環 |
| 生態鏈 | M2 | 有群體穩態，但未必具 formal packet/log |
| 基因序列鏈 | M2 | 序列與可證性強，但回寫 / 重建鏈未必以治理形式存在 |
| 邏輯鏈 | M2 | 具前提與回證，但未必具治理與回寫程序 |
| 架構鏈 | M3 | 可對位、可承接，已有閉環潛力 |
| 依存鏈 | M2 | 為母鏈，需依具體外化程度判定 |
| 回寫鏈 | M4 | 只要 packet/log/rollup 齊備即可進閉環 |
| 審查鏈 | M3 | 七判程序可形成穩定成立語言 |
| 重建鏈 | M4 | recovery / snapshot / audit / re-entry 齊備即可 |

## 4. 升階規則
### M0 -> M1
- 必須有最小成立條件描述

### M1 -> M2
- 必須對位至少一個主盤位、一個七位主對位、一個主要窗口

### M2 -> M3
- 必須可輸出結果包
- 必須具 source_ref / current_state / responsible_ref / next action

### M3 -> M4
- 必須有 packet + log + rollup + re-entry

### M4 -> M5
- 必須有 recovery point / snapshot / audit trail
- 必須能在失配後重建並再入鏈

## 5. 使用方式
- 用於判斷某鏈目前只是概念、分類，還是已進入可運行層
- 用於比較不同鏈型目前在靈絡中的成熟度
- 用於安排下一輪補骨優先順序

## 6. 收斂語句
鏈的成熟，不看它名字有多大，而看它是否能被證明、能被承接、能被回寫、能被重建；靈絡要做的，正是讓世界各種已命名與未命名之鏈，逐步升格為可治理鏈生態。
