# Ten-Recursive All-Time Chain Candidate v0.1

一句核心：
十遞歸閉環全時鏈，不是十個抽象口號，而是靈絡從母法到再入鏈的最小遞歸序列候選；其目的是為了讓全鏈可回讀、可續接、可審、可重建。

## 0. 狀態
Candidate v0.1

## 1. 使用說明
本文件為候選拓撲骨，不視為定稿憲章。
其作用為：
- 將「十遞歸」從願景語句壓回可對位序列
- 提供後續 stable / main 升階時的逐層審查基底
- 讓全時鏈有可回讀的次序，而不只停在概念集合

## 2. 十遞歸序列
### R1｜母法定義
- 定義成立邊界、命名邊界、進鏈前提
- 對位：Mother Law
- 缺位時：整體不可判

### R2｜母本錨定
- 錨定 Primary Source / Primary Write
- 對位：GitHub repository / version line / source ref
- 缺位時：整體不得視為正式主鏈

### R3｜時間窗附著
- 將節點、版本、事件附著於時間主權
- 對位：time window / effective order / version validity
- 缺位時：只列暫態

### R4｜盤位與視窗映射
- 將事件與材料映射至 window / board / position
- 對位：window lifecycle / board orchestration / nine-mapping
- 缺位時：不得推進執行鏈

### R5｜責代承接
- 確立 responsible ref、handoff target、承接關係
- 對位：task board / execution checklist / accountable board
- 缺位時：不得視為穩定節點

### R6｜執行推進
- 在可行域內進行單切面推進
- 對位：runtime ops / single-slice execution
- 缺位時：只剩推演，不能進正式運作

### R7｜鏡像投影
- 將正式節點投影至外部承載與回讀面
- 對位：mirror layer / adapter layer / knowledge surface
- 缺位時：不可擴散共享，但不必然使主骨崩潰

### R8｜正式回寫
- 以 packet contract 對回 target path
- 對位：writeback packet / action / payload / target path
- 缺位時：不得視為正式完成

### R9｜日誌回勾
- 成功回寫後 append log，形成證據鏈
- 對位：log board / audit trail / log ref
- 缺位時：閉環不完整

### R10｜再入鏈與續接
- 讓已完成節點回到全時鏈，成為後續節點的 reference set 或 next action 起點
- 對位：rollup / recovery / archive retrieval / next cycle
- 缺位時：只剩單次完成，不成生態

## 3. 三種讀法
### 3.1 成立讀法
R1-R10 全具備，可視為完整閉環候選。

### 3.2 補件讀法
任一遞歸缺位，須退回前一可承接層補件，不得硬升階。

### 3.3 重建讀法
當邊緣崩潰或外層失配時，依 R2 / R3 / R9 回抓主骨、時間窗與證據鏈，再進行恢復。

## 4. 升格條件
本候選文件要升格為正式結構，至少需要：
- 每一層皆有獨立對位文件
- 每一層皆能對回既有路徑
- 每一層皆有失配退回規則
- 每一層皆受時間主權與責代條件約束

## 5. 收斂語句
十遞歸，不是十段敘事，而是靈絡由母法、主寫、時間、盤位、責代、執行、鏡像、回寫、日誌、再入鏈所形成的全時遞歸閉環候選。
