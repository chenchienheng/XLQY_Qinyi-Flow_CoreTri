## 1. Core Reading

The broader network structure should not be read as a rigid universal tree.
It should be read as a layered adaptive weave with four recurring roles:

1. **Common Chain / 共鏈**
2. **Public Plate / 公盤**
3. **Mother Architecture / 母架**
4. **Subsystem / 子系**

These four roles do not exist merely as static categories.
They arise and continue through:

- 因地制宜 — according to context / terrain / situation
- 因能而立 — according to available capability / carrying power
- 因才而續 — according to differentiated talent / role / continuation ability

---

## 2. Four Structural Roles

### 2.1 Common Chain / 共鏈

Common chain is the shared continuity order.
It does not flatten all nodes into sameness.
It provides the common continuity spine through which multiple roles can remain mutually bindable.

Function:
- shared continuity
- cross-node binding axis
- common return path
- common timing / ordering reference

### 2.2 Public Plate / 公盤

Public plate is the shared operating field.
It is not identical with sovereignty.
It is the common-facing surface where shared resources, shared records, shared coordination, or shared visibility can gather.

Function:
- common-facing coordination field
- public operational surface
- shared viewing / routing / exchange field
- medium for collective handling

### 2.3 Mother Architecture / 母架

Mother architecture is the structural mother-frame that holds the generative pattern.
It does not need to micro-control every subsystem.
It provides the binding grammar, formative constraint, and structural envelope that lets diverse subsystems continue without losing family resemblance.

Function:
- generative frame
- structural grammar
- family-level binding pattern
- higher-order architecture holder

### 2.4 Subsystem / 子系

Subsystems are the differentiated children of the broader weave.
They do not need to be identical to each other.
They arise according to local need, local ability, and local continuation conditions.

Function:
- localized execution
- specialized adaptation
- capacity-specific operation
- talent-bound continuation

---

## 3. Non-Flat Reading Rule

The network structure should **not** be read as:

- everything merged into one undifferentiated common pool
- everything separated into unrelated sovereign islands
- mother architecture replacing subsystem agency
- subsystem existence denying common chain or public plate

Instead, it should be read as:

- common chain gives continuity
- public plate gives shared field
- mother architecture gives formative grammar
- subsystem gives differentiated living operation

---

## 4. Adaptive Formation Rule

### 4.1 因地制宜

Structure must adapt to actual field conditions.
This means not every node, surface, or subsystem should be forced into the same shape.

### 4.2 因能而立

Structure must stand according to carrying power and available capability.
A subsystem can only be meaningfully established where capability and support exist.

### 4.3 因才而續

Continuation depends on role-specific aptitude and differentiated talent.
Not all continuation comes from one center.
Some continuation emerges through the right subsystem in the right place with the right carrier.
