## 1. Five-Field Rule

Every name must carry five fields before use:

```yaml
Name_Check:
  name:
  layer:
  purpose:
  forbidden_misuse:
  status:
```

If any field is missing, the name remains unclassified and should not be promoted.

---

## 2. First Question Rule

Before accepting or publishing any name, ask:

```yaml
Naming_Classification_Questions:
  ontology_type:
    ask: is this body, interface, protocol, flow, document, case, person, valence, or historical shell?
  exposure:
    ask: can this be public-facing, internal-only, private-boundary, or support-pack-only?
  runtime_risk:
    ask: could this make readers think something is deployed, approved, executable, or externally authorized?
  private_relation_risk:
    ask: could this systematize a private relationship or living person?
  sovereignty_risk:
    ask: could this make a tool, platform, model, or carrier look like the system center?
```

## 15. Public Exposure Rule

Before moving any name into a public-facing document, check:

```yaml
Public_Exposure_Check:
  can_explain_without_private_people: true_or_false
  does_not_imply_runtime: true_or_false
  does_not_imply_approval: true_or_false
  does_not_make_tool_sovereign: true_or_false
  does_not_publish_private_mapping: true_or_false
  source_lineage_clear: true_or_false
```

If any value is false, keep the name internal or private-boundary.
