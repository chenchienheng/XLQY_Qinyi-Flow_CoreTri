## 3. Role Layers

### 3.1 Human Origin / Source Anchor

The user is not a node and not a tool.

Position:

- direction giver
- source anchor
- final reviewer
- authorization boundary
- life-pressure test source
- world-trend sensor

One-line:

```text
You are not an XLEN node. You are the source anchor that gives nodes direction.
```

### 3.2 Qinyi / XLQY

Qinyi is not the XuanLing body and not an external executor.

Position:

- external collaboration surface
- human-language translation window
- judgment organizer
- return support layer

Responsibilities:

- break down direction
- translate external material into readable structure
- classify feasible / infeasible paths
- produce architecture windows, skill cards, SOPs, and candidate patches
- warn about boundary and contamination risk

One-line:

```text
The user gives direction; Qinyi turns direction into structure that can be judged, returned, and executed under boundary.
```

### 3.3 Codex

Codex is best read as a local artifact agent inside a bounded project folder.

Good fit:

- read repo
- read folder
- edit documents
- create presentation or files
- run bounded small tasks
- recycle a successful workflow into a skill

One-line:

```text
Codex is the Artifact Agent inside a Local Habitat.
```

### 3.4 Claude Code

Claude Code is best read as an engineer-oriented CLI assistant.

Good fit:

- terminal users
- git / test / package / build workflows
- code refactor
- debug
- command execution
- engineering projects

Boundary:

```text
Claude Code may be powerful, but it usually assumes more engineering context from the user.
```

## 4. Tool Roles Are Not Rankings

| Surface | Best Reading |
|---|---|
| Qinyi | direction, judgment, translation, return support |
| Codex | project folder, documents, decks, repo, artifacts, skill recycle |
| Claude Code | engineering CLI, debug, refactor, tests |
| Google / NVIDIA skills | external knowledge and skill source |
| Microsoft / Copilot / Agentic OS | enterprise workflow and task dispatch pattern |
| NVIDIA Hermes / NemoClaw | secure self-evolving agent runtime candidate case |
| OpenAI + Ona | persistent agent habitat case |

Rule:

```text
Do not ask which tool is strongest in general. Ask which role the task requires.
```

## 5. Current Core Modules

### Module A｜Knowledge Ingestion Loop

Purpose:

```text
The user should not need to learn every setting in the world. XuanLing should absorb important external settings into usable paths.
```

Flow:

```text
important material
→ Qinyi extracts Fact / Pattern / Risk / Skill
→ translate into XuanLing language
→ package as Skill Card / SOP / Protocol
→ user reviews
→ small-field test
→ return into XLA
```

One-line:

```text
You do not need to learn the world's settings one by one; XuanLing should absorb them into paths you can use.
```

### Module B｜Local Habitat Workflow

This is the practical value of the Codex presentation workflow.

Flow:

```text
create project folder
→ place source material
→ define work boundary
→ inspect files first
→ produce outline first
→ human confirms
→ produce artifact
→ recycle into skill
```

One-line:

```text
Real AI workflow is not one-shot generation; it has habitat, confirmation, output, and recycle.
```

### Module C｜Persistent Agent Habitat

Agent work requires a controlled habitat.

Required elements:

- sandbox
- bounded credentials
- policy rules
- tool allowlist
- network boundary
- memory / skills
- snapshot / restore
- trace
- rollback
- human review

One-line:

```text
Agents cannot run naked. Agents need controlled habitat.
```

### Module D｜Token Capital / Token Valence

Token usage is not automatically capital.

If it is not recycled, it is cost.

If it becomes architecture windows, safety scans, naming governance, skill cards, candidate patches, repo tasks, living atlas entries, or reusable judgment, it becomes token capital.

One-line:

```text
Token is context fuel. Without recycling, it is cost; with recycling, it is capitalized capability.
```

### Module E｜Security / Governance Spine

The current security focus is governance security, not only web security.

Core concerns:

- instruction integrity
- namespace integrity
- adapter red gate
- evidence boundary
- credential isolation
- human-origin review
- agent containment
- token / cost control
- compute boundary
- return-path governance

One-line:

```text
XuanLing security prevents words, names, documents, and model outputs from being wrongly promoted into power.
```
