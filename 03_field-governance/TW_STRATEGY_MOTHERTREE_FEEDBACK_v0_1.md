# Taiwan Strategy MotherTree Feedback v0.1

## Purpose

Record feedback from the Taiwan Strategy branch after receiving the MotherTree reframe task.

This is a non-core return packet. It records branch learning, boundary correction, and next actions without writing MotherTree core or private dependency chains into the company-facing strategy branch.

## Core Feedback

The MotherTree reframe helped Taiwan Strategy move from data patching into a returnable branch chain:

```text
Market point cloud -> Knowledge Molecules -> Strategic Relationship Matrix -> Shimizu entry point -> GM decision map
```

## What Changed

```yaml
Task_Framing:
  Before:
    - market data, vendors, rankings, and slides were easily mixed into one layer
  After:
    - Capture / Converge / Harden stages are separated
    - Capture may keep point-cloud candidates without deleting weak evidence
    - Converge sorts Role / Vector / Facility / Dependency
    - Harden is required before formal narrative use

Evidence_Boundary:
  Before:
    - capability, investment, and media signals could be over-read as usable conclusions
  After:
    - each claim must link to Source_ID
    - each conclusion must include Can_Support / Cannot_Support
    - Pending cannot enter formal main text

No_Delete_Principle:
  Before:
    - candidate vendors and market nodes could be thinned too early for table neatness
  After:
    - weak data is not deleted
    - weak data is marked Radar / Strategic Signal / Pending / Downgraded

Relationship_Matrix:
  Before:
    - work could stop at vendor list or market list
  After:
    - Strategic_Relationship_Matrix becomes the convergence object
    - fields include Parent_Group, Role, Market_Vector, Facility_Type, Dependency_Link, Interface_Risk, Shimizu_Relevance, and Possible_Entry_Point

GM_View:
  Before:
    - output risked becoming data stacking instead of decision structure
  After:
    - final output converges into GM View
    - 3-5 high-confidence opportunity lanes are separated from Radar nodes
    - Go / No-Go / Conditional Pass gates are retained

Branch_Boundary:
  Before:
    - high-level architecture could be inserted into the company database body
  After:
    - this is positioned as a Taiwan Strategy branch task specification
    - MotherTree structure is referenced without writing core, invariant nucleus, or private dependency chains into the company ERP body

System_Learning:
  Before:
    - the branch could jump steps, overbuild architecture, or delay urgent delivery
  After:
    - urgent deliverables go first
    - DB writeback follows
    - branch scope is located before expansion
```
