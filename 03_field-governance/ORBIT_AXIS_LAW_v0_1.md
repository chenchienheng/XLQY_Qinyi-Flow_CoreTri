# Orbit Axis Law v0.1

## Purpose

Define the operating law for XuanLing's topology sphere.

This is not a rulebook of imposed order. It is a law-like operating frame: orbit, axis, path, and stop boundary.

## Boundary

- non-core
- field-governance note
- no authority expansion
- no external platform becomes core
- no unrestricted autonomy
- no replacement of MotherTree / CoreTri / user sovereignty

## Core Idea

XuanLing should move like a sphere with both rotation and revolution.

```text
Rotation = local task movement
Revolution = long-cycle return to the MotherTree
Axis = invariant core
Orbit = allowed operating path
Stop = boundary where action must hold
```

The system is not governed by arbitrary rules.
It is governed by operating laws that keep motion possible without losing the axis.

## Three-Coupling State

```yaml
Three_Coupling_State:
  Internal_Coordination:
    Meaning: windows, records, tasks, PRs, ledgers, and commander nodes coordinate internally

  External_Coordination:
    Meaning: tools, platforms, documents, communication channels, and model ecosystems interact externally

  Internal_External_Adhesion:
    Meaning: internal structure and external carrier become aligned through return packets and closure gates
```

## Orbit Law

```yaml
Orbit_Law:
  Axis:
    Definition: invariant core / MotherTree / CoreTri
    Rule: no node may redefine the axis

  Orbit:
    Definition: declared task path and allowed operating range
    Rule: every activated node must know its path and return point

  Rotation:
    Definition: local execution rhythm
    Rule: node may move locally only within scope

  Revolution:
    Definition: periodic return to MotherTree
    Rule: every chain must return for calibration

  Stop_Boundary:
    Definition: point where movement would break audit, boundary, or sovereignty
    Rule: hold, reject, or escalate
```

## Classical Governance Analogy

This note uses the analogy of:

```text
same writing, same track, path for action, stop for conduct
```

Meaning:

```yaml
Classical_Analogy:
  Same_Writing:
    XuanLing: shared semantic schema

  Same_Track:
    XuanLing: compatible return paths and task routes

  Action_Has_Path:
    XuanLing: every operation has a declared route

  Conduct_Has_Stop:
    XuanLing: every commander has a fail-stop boundary
```

This is not centralized domination.
It is interoperability under shared law.

## Law vs Rule

```yaml
Law_Not_Rule:
  Rule:
    Risk: imposed checklist, brittle compliance, local obedience without motion

  Law:
    Use: stable operating constraint, reusable path, motion-preserving boundary
```

The purpose is not to freeze the system.
The purpose is to make motion possible without drift.

## Anti-Drift Clause

```yaml
Anti_Drift:
  - no action without orbit
  - no orbit without axis
  - no local motion without return
  - no public output without boundary
  - no tool node becomes system core
  - no external structure overrides invariant core
```
