## 2. Why This Matters

This was not merely more Codex usage.

It was a first pressure test of XuanLing / Qinyi grammar on a workspace agent.

Codex is not controller, XuanLing body, Qinyi replacement, or merely coding executor.

More precise positioning:

```text
Codex = bounded workspace support-cell / local project worker
```

Codex may handle project folders, repos, documents, artifacts, review packs, and build-ready packs only under:

- LOR
- Gate
- Return Packet
- Artifact Verification
- Cost Gate
- Stop Condition

---

## 3. Core Interpretation

Without contract, Codex may become:

- verbose
- multi-branch
- high-token
- hard to verify
- unclear in status
- prone to treating artifact output as deliverable
- prone to treating draft as build-ready
- prone to treating review pack as per-source revised pack

Under XuanLing grammar, Codex can become governable when the route is:

```text
Source Check
→ Outline / Plan
→ Human Confirm
→ Artifact Output
→ Artifact Verification
→ Return Packet
→ Resume Card / Next LOR
```

This means Codex may be absorbed into XLA workspace-agent logic only as a candidate support cell, not as controller or runtime.

---

## 4. What Qinyi Learned

1. Codex needs Local Habitat.
   Project Folder / Repo / Local Workspace are carriers with different risks. OneDrive, local path, and GitHub repo must not be collapsed.

2. Codex needs Deliverable Contract.
   Without contract, it can produce attractive but unverifiable output.

3. Codex needs Artifact Verification.
   Artifacts need source mapping, manifest, diff summary, unresolved risk, and review need.

4. Codex needs Cost Gate.
   High token without return is burn. High token with structure return may become Token Capital.

5. Codex needs Stop Condition.
   Ability does not equal authorization to continue.

6. Codex needs Return / Resume.
   Long tasks need next-round continuity, not just summary endings.

## 5. Architecture Additions

### 5.1 Workspace Support Cell

Candidate support cells:

- PC_Diagnostic / Local Habitat Health
- Codex_Settings
- Artifact_Review
- OLD_Review / OneDrive_Metadata_Review
- Flow_Cell_Foundation
- Build_Ready_Check
- Security_Context_Assist

Support cells maintain mainline stability. They do not replace the mainline.

### 5.2 Local Habitat

Local Habitat should check:

- whether the path is inside OneDrive
- whether sync is active
- whether metadata conflicts exist
- whether local workspace is confused with source of truth
- whether Defender / indexing / browser / disk I/O will amplify cost
- whether artifact generation is appropriate
- whether PC Diagnostic is needed first

If Local Habitat is unhealthy, avoid long or high-cost tasks.

### 5.3 Artifact Mode

Candidate artifact modes:

- Review Pack
- Per-Source Revised Pack
- Build-Ready Pack
- Editable Route
- Visual Route
- QA Report
- Artifact Manifest
- Resume Card

Hard separations:

```text
Review Pack ≠ Per-Source Revised Pack
Build-Ready Draft ≠ Runtime Built
Editable Route ≠ Visual Route
Artifact Output ≠ Verified Deliverable
Summary ≠ Closeout
```

### 5.4 Codex Workflow

Candidate workflow:

1. Project Folder = Local Habitat
2. Source Check before generation
3. Deliverable Contract before artifact
4. Candidate output, not source overwrite
5. Artifact verification after creation
6. Human review before next escalation
7. Closeout must include Next_LOR_Ready or Resume_Card

### 5.5 Return Packet Extension

Suggested fields:

- Carrier
- External_Effects
- Artifact_Diff_Summary
- Stop_Condition
- Next_LOR_Ready
- Resume_Card
- Cost_Gate
- Artifact_Mode
- Review_Need

## 6. Candidate Valences Added

### Workspace Agent Valence

Ask which carrier Codex is working in: conversation, repo, local folder, synced cloud, AGENTS.md, Skill, Flow, M365, or GitHub.

### Local Habitat Health Valence

Ask whether the workspace is healthy: PC slowdown, OneDrive sync, Defender scanning, disk I/O, browser RAM pressure.

### Artifact Verification Valence

Ask whether output has source, version, manifest, diff, risk, acceptance criteria, and return path.

### Cost Gate Valence

Ask whether token / tool / runtime cost has budget, retry limit, stop condition, and recycled output.

### Resume Valence

Ask whether interruption can resume with Resume Card, unresolved risks, next step, and human review requirement.
