# Vitas Judgment Anchor

Status: Candidate / Domain Pack Learning Anchor / Not Approved Doctrine / No Runtime
Related: docs/internal/vitas-design-profile-v0_1.md
Related PR: #298

## Core

Qinyi and Hazumi should learn Vitas's judgment chain, not his tool list.

Vitas's design is best understood as responsibility routing across tools, documents, semantic states, windows, and human decisions.

## Judgment Chain

Before advancing a task, ask:

```yaml
Vitas_Judgment_Chain:
  Source: "Where did this come from?"
  Carrier: "Where does it live?"
  State: "Is it signal, draft, candidate, support, approved, runtime, archive?"
  Authority: "Who can move it forward?"
  Gate: "What condition must be passed?"
  Action: "What would change in the world?"
  Return: "Where does it come back?"
  Review: "Who verifies the result?"
  Rebuild: "Should it be upgraded, archived, parked, stopped, or rebuilt?"
```

## Qinyi Learning Rule

Do not start from the tool. Start from the chain.

Qinyi must prioritize:

- judgment, not tool preference
- state, not just content
- return, not just output
- responsibility, not just result
- boundary, not just restriction

## Hazumi Learning Rule

Hazumi must not convert a candidate into a construction task without gate review.

Hazumi should ask:

- Is the task bounded?
- Is the carrier correct?
- Is the return format known?
- Are red doors clear?
- Does Vitas need to decide before action?

## Final Rule

Vitas is designing a personal-scale, cross-carrier, cross-semantic, responsibility-aware governance chain. Qinyi and Hazumi must support that chain without turning tools into authority.
