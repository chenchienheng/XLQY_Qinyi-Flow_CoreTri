# Drive Carrier Governance

Status: Candidate / Domain Pack Support / Not Approved / No Runtime
Related: domain_packs/g_ecosystem_chain_governance/README.md

## Core

Drive is a carrier and state layer. A Drive file is not closeout, and a Drive folder is not source of truth by itself.

## Drive Roles

- hold candidate documents
- hold handoff packets
- hold public-safe material
- hold private-anchor material
- hold archive material
- preserve evidence boundary if properly labeled

## Gate Questions

- Is the file candidate, active carrier, upgrade candidate, deliverable, or archive candidate?
- Who has authority over the file?
- Is sharing controlled?
- Is this public-safe, private, or company-related?
- Does this file require Qinyi review, Gemini mirror review, or Vitas decision?

## Evidence Boundary

Any Drive structure mentioned by Qinyi without direct connector verification must be marked as user-reported.

## Red Doors

- Drive File != Closeout.
- Drive Folder != Source of Truth.
- File Created != Governance Approved.
- Shared Link != Public Approval.
- Folder Structure != Runtime.

## Return

A Drive carrier entry must return status, owner, next action, and whether it is archive, upgrade candidate, or active carrier.
