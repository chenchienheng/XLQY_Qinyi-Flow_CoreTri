## Core

M ecosystem should be connected into the same XuanLing chain, but not as a separate sovereign logic. It is another carrier ecology where the same governance grammar applies.

```text
Source → Carrier → Authority → Gate → Action → Return → Rebuild
```

M is not only company M365. It has personal and enterprise modes. The enterprise mode has stronger red lines, but the bottom grammar remains the same.

## Two Modes

```yaml
M_Ecosystem_Modes:
  Personal_M:
    nature: "personal Microsoft cloud carrier"
    possible_surfaces:
      Outlook: "signal gate / mail carrier"
      OneDrive: "document carrier / state layer"
      Calendar: "time gate"
      To_Do: "task reminder candidate"
      OneNote_or_Loop: "note / knowledge carrier if available"
    authority: "Vitas personal authority"
  Company_M:
    nature: "enterprise permissioned operational carrier"
    possible_surfaces:
      Outlook: "formal company communication"
      Teams: "collaboration and decision trace candidate"
      SharePoint: "company evidence carrier"
      Lists: "structured work unit carrier"
      Power_Apps: "manual build UI candidate"
      Power_Automate: "runtime candidate only after approval"
    authority: "company / tenant / owner / admin / human gate"
```

## M Chain Flow

```text
Outlook / Teams Signal
→ Qinyi Judgment
→ OneDrive / SharePoint Carrier
→ M-side Mirror or Internal Review
→ Calendar / To Do Time Gate
→ Return Packet
→ Vitas or Company Authority Decision
→ Archive / Upgrade / Schedule / Manual Build
```

## Red Doors

- M ecosystem connected != M365 authorized.
- Personal M != Company M.
- Outlook category != responsibility.
- OneDrive file != closeout.
- SharePoint file != company decision.
- Teams discussion != formal approval.
- Calendar event != execution.
- To Do item != responsibility accepted.
- Manual build guide != system built.
- Power Automate design != Flow runtime.
- Copilot / assistant output != company position.

## Final Rule

M ecosystem can be linked into the XuanLing chain now as a domain pack. Company-side action remains gated. The goal is not to copy G into M, but to let the same trans-domain governance grammar appear inside M without moving data, authority, or responsibility across the wrong boundary.
