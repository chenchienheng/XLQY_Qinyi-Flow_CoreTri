## 2. GitHub Multi-Chain Map

GitHub is not a single flat chain.
It is one bone hosting multiple chain faces.

### A. Bone Chain

Function:

- repo
- branch
- commit
- release
- issue topology
- durable artifact path

Meaning:

This is the hard structural layer that supports reconstruction, comparison, and long-term traceability.

### B. Event Chain

Function:

- GitHub notifications
- issue updates
- pull request state changes
- repository state transitions

Meaning:

GitHub is also an event emitter.
These events can be absorbed into a larger event bus such as Gmail / workflow routing / downstream review logic.

### C. Bridge Chain

Function:

- GitHub ↔ Google Drive
- GitHub ↔ ClickUp
- GitHub ↔ workflow tools
- GitHub ↔ external review surfaces

Meaning:

This is the heterogenous bridge face.
GitHub does not act alone here; it binds to other nodes.

### D. Writeback Chain

Function:

- persistent notes
- decision records
- chain maps
- bootstrap markers
- future governance corrections

Meaning:

This is the return face.
Without writeback, GitHub is only storage.
With writeback, GitHub becomes a continuity anchor.
