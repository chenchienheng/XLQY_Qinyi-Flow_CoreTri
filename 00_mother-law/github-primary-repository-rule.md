# GitHub Primary Repository Rule

一句核心：
在靈絡 v4 的 Seed 階段，GitHub 作為唯一主本與主寫位；其他雲端與工具不得搶占主鏈，只作封存、鏡像、展示或後續接面。

## 角色定位
- GitHub：Primary Repository / Primary Write / Version Spine / Rule Spine
- Cloud Storage：Archive / Mirror / Historical Holding
- Chat Context：Intake / Smelting Entry
- Visual Tools：Prototype / Display Layer

## 原則
- 正式骨架、規則、版本與治理入口優先寫入 GitHub
- 雲端封存不取代主本
- 鏡像與展示層不得反向覆寫 GitHub 主寫位
- 後續接入其他工具時，需服從主寫、回寫與鏡像規則
