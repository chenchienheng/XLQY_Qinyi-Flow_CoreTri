# Five Family Digital Ecosystem Pillars v0.1

一句核心：
四盤成形後，翾靈的下一層不是工作流，而是五柱家族生態：Google、Microsoft、Apple、Adobe、AI；五柱不是要被崇拜或對抗，而是要被映射成數位世界的五大入口柱，最後回到靈絡主骨統一裁定、驗證與回流。

## 0. 狀態
Draft v0.1

## 1. 前提
四盤提供同步基礎：
- 內生主枝幹網
- 外部證據追蹤盤
- 排程登錄盤
- 靈絡節點封包盤

五柱則提供數位世界入口面。

四盤回答：
- 主骨如何同步
- 節點如何成包
- 外證如何追蹤
- 時間如何觸發

五柱回答：
- 數位世界如何落入靈絡場
- 各大家族如何被降格成可映射生態柱
- 翾靈如何從工具流升到世界數位場義

## 3. 五柱共同降格規則
五柱不得以品牌身份進主骨。
五柱只以以下身份進入：
- source provider
- state surface
- action executor
- result producer
- mismatch detector
- responsibility holder
- time-order signal
- writeback target

## 4. 五柱映射等級
- FP0｜External Brand：只是外部品牌，未映射
- FP1｜Observable Surface：可觀測某個狀態
- FP2｜Mappable Interface：可抽 source / state / result
- FP3｜Routable Ecosystem Node：可分派到窗口
- FP4｜Writeback Candidate：具回寫或更新可能
- FP5｜Lingluo Pillar Node：可驗證、可回寫、可回流、可回 00

## 5. 五柱與四盤的關係
| Four Board | 五柱用途 |
|---|---|
| 內生主枝幹網 | 判斷五柱落在哪個主枝幹 |
| 外部證據追蹤盤 | 追蹤五柱是否真的留下外證 |
| 排程登錄盤 | 決定五柱何時被低載喚醒或事件觸發 |
| 靈絡節點封包盤 | 將五柱輸出壓成 node packet |

## 6. 為什麼不是 workflow
workflow 只是工具流程。
五柱是數位世界入口面。

workflow 會被平台吞掉。
五柱若被靈絡映射，會回流成：
- 規則
- 節點
- 外證
- 路由
- 審計
- 可替換模組

## 7. 禁止事項
- 禁止用品牌崇拜取代映射
- 禁止把五柱任何一家視為主骨
- 禁止把 workflow automation 視為翾靈完成態
- 禁止無 evidence_ref 卻宣稱打入五柱
- 禁止五柱輸出跳過 00 裁定

## 8. 收斂語句
五柱不是要被征服，而是要被降格成靈絡可映射、可驗證、可回寫、可回流的數位入口柱；當 Google、Microsoft、Apple、Adobe、AI 都能以節點形式進入同一主骨時，翾靈才真正開始打入整個數位世界。
