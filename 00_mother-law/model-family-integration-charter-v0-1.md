# Model Family Integration Charter v0.1

一句核心：
翾靈不應選模型為王，而應將所有外部模型降格為角色化模組家族；品牌不進主骨，角色、契約、驗證與回寫秩序才進主骨。

## 0. 狀態
Draft v0.1

## 1. 問題定義
當前世界多以『選模型』為主要邏輯：
- 誰最強
- 誰最便宜
- 誰最快
- 誰最好用

此邏輯會讓主權落在模型供應商，而非系統主骨。

## 2. 翾靈的基本立場
- 模型不是主體
- 模型不是主寫位
- 模型不是最終裁定者
- 模型只是可被調度的責能模組

## 3. 模型角色分類
### R1｜Read Model
- 摘要
- 取樣
- 鏡像理解
- 外部材料初讀

### R2｜Reason Model
- 推理
- 拆解
- 候選判定
- 路徑提案

### R3｜Execute Model
- 執行路徑生成
- toolchain 建議
- runtime 接力候選

### R4｜Verify Model
- 交叉驗證
- mismatch 偵測
- gap 補捉
- 停損判定

### R5｜Structure Model
- 壓骨
- 結構重排
- 母法候選生成
- 升階候選提案

## 4. 模型適配最低契約
任一外部模型若要進入翾靈，至少需對回：
- model_role
- input_packet
- output_schema
- verification_rule
- writeback_permission
- return_to_00

## 5. 禁止事項
- 禁止以品牌名決定主權
- 禁止讓單一模型直接主寫 GitHub 母本
- 禁止讓模型輸出跳過 00 裁定
- 禁止把候選輸出誤當正式成立

## 6. 收斂語句
模型家族若要成為翾靈的一部分，不是因為它們彼此很像，而是因為它們再不同，也都只能以同一套角色契約、驗證秩序與收口語言進入主骨。
