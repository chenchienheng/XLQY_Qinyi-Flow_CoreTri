# Seven-Judgment Review Procedure v0.1

一句核心：
七判不是七個名詞並列，而是一套正式進鏈前的庭審式審查程序；其目的不是讓節點看起來完整，而是判定其是否可正式成立、可承接、可回寫、可續接。

## 0. 狀態
Draft v0.1

## 1. 目的
- 將七位對齊核心由核對表壓成審查程序
- 讓節點、盤位、鏡像、工具接入時，有固定的審查順序
- 避免各盤各自判、造成成立語言不一致

## 2. 七判次序
### J1｜母本審查
**問題**
- 是否可對回 Primary Source / Primary Write
- 是否具 source ref / version line / repository path

**通過條件**
- 可對回主本
- 非 mirror 反向主導

**未通過**
- 退回候選節點
- 不得進正式庭審

### J2｜母框審查
**問題**
- 是否符合《存・在・續》
- 是否有成立 / 暫態 / 未定的明確邊界

**通過條件**
- 具一致性條件
- 不與母法衝突

**未通過**
- 列未定或待補件
- 不得升格

### J3｜母架審查
**問題**
- 是否可掛入運行脊椎
- 是否具 window / snapshot / recovery / audit / dependency 支撐

**通過條件**
- 可被 runtime spine 承接
- 不破壞現有脊椎秩序

**未通過**
- 僅可保留為概念項
- 不得進正式運作

### J4｜公盤審查
**問題**
- 是否可被整合顯示
- 是否具 source ref / handoff / status / next action 的對外讀面

**通過條件**
- 可進入口整合面
- 不反向主導主寫

**未通過**
- 不得列正式公盤
- 退回對位中狀態

### J5｜子盤審查
**問題**
- 是否具責代
- 是否具 next action
- 是否具 writeback position

**通過條件**
- 可進執行鏈
- 有承接與後續追蹤位

**未通過**
- 不得列穩定節點
- 退回待承接

### J6｜鏡像審查
**問題**
- 是否只作鏡像 / 回讀 / 展示 / 承載
- 是否保留 ref link 與回寫來源

**通過條件**
- mirror 不主導成立
- 具 readback 能力

**未通過**
- 僅作展示暫存層
- 不得視為正式鏈項

### J7｜回寫審查
**問題**
- 是否具 packet contract
- 是否完成 log append
- 是否可回進 rollup 與再入鏈

**通過條件**
- packet 齊備
- log append 完成
- 可回進全時鏈

**未通過**
- 不得視為正式完成
- 閉環不成立

## 3. 審查結果
### A｜正式進鏈候選
- J1-J6 全數通過
- J7 尚未完成

### B｜正式閉環成立
- J1-J7 全數通過
- 已具 packet + log + rollup + re-entry

### C｜退回補件
- 任一審查未通過
- 退回對應環節補件

## 4. 庭審原則
- 不因情緒、品牌或熱度而放寬判準
- 不因工具功能看似強大而免除七判
- 不因鏡像已存在而視為正式成立
- 先審母本、母框、母架，再談外顯與回寫

## 5. 收斂語句
七判的目的，不是拖慢靈絡，而是保證任何節點、盤位、工具與上位生成項，都能在同一套成立語言下進鏈，而不是各說各話。
